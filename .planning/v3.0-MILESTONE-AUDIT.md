### Integration Bugs Found

**BUG 1 — All-zero Claude context (Priority 1, affects NLQ-03/04/09)**

`/src/components/data-display.tsx` lines 397-429: `QuerySearchBarWithContext` reads wrong Snowflake column names:
- `r.PENETRATION_RATE` → should be `PENETRATION_RATE_POSSIBLE_AND_CONFIRMED`
- `r.COLLECTION_RATE_6MO` → should be `COLLECTION_AFTER_6_MONTH`
- `r.COLLECTION_RATE_12MO` → should be `COLLECTION_AFTER_12_MONTH`
- `r.TOTAL_COLLECTED` → should be `TOTAL_COLLECTED_LIFE_TIME`
- `r.TOTAL_PLACED` → should be `TOTAL_AMOUNT_PLACED`

All resolve to `undefined`, producing `Number(undefined) || 0 = 0`. Fix: replace manual stats construction with `computeKpis(rows)` from `@/lib/computation/compute-kpis`.

**BUG 2 — Anomaly badge missing at root level (Priority 2, affects AD-07)**

`/src/lib/columns/root-columns.ts`: `buildRootColumnDefs()` does not include `anomalyStatusColumn`. The column is prepended in `buildColumnDefs()` (used at partner/batch levels) but the root-level override bypasses it. Fix: import and prepend `anomalyStatusColumn` in `buildRootColumnDefs()`.

## User-Reported UI Issues (2026-04-14)

Issues observed during manual testing, to be included in gap closure plan:

### UI-01: Curve chart tooltip stuck on first batch
Clicking different batch lines on the collection curve chart does not update the tooltip. It appears locked to whichever batch was first selected. Likely a stale state issue in `useCurveChartState` or `CurveTooltip` — the solo/active batch state may not be updating the tooltip's data source.

### UI-02: Hide graph icon missing + partner-level table rows hidden on laptop
The toggle to collapse/hide the chart is not visible. On a laptop screen at partner drill level, the chart takes up enough space that the data table rows are pushed below the fold with no obvious way to dismiss the chart. Need a visible collapse/minimize control for the chart area, and possibly a more compact default chart height at partner level.

### UI-03: Partner comparison button unclear
The "partner comparison" button's behavior is not self-explanatory. The view shift it triggers is ambiguous — users can't tell what changed or what they're looking at. Needs clearer labeling, a visual transition, or contextual explanation of what the comparison mode shows.

### UI-04: View switch doesn't update chart
Switching metric views (e.g., selecting "Outreach" from the view selector) does not update the collection curve chart to show the corresponding metric. The chart remains on recovery data regardless of which view is active. The chart's metric source needs to be reactive to the active view/metric selection.

## Data Loading Issues (2026-04-14)

Three new account JSON files in `src/lib/static-cache/` (downloaded 2026-04-13) fail to load correctly:

### DATA-01: Missing ACCOUNT_PUBLIC_ID column
Files `accounts-affirm-afrm_dec_25_pri.json`, `accounts-affirm-afrm_nov_25_pri.json`, and `accounts-aff-aff_feb_26_lto_tertiary.json` have 18 columns instead of 19. The `ACCOUNT_PUBLIC_ID` column expected by `account-definitions.ts` is absent. Batch-level drill-down breaks when the table tries to access this field.

### DATA-02: Empty strings instead of NULLs
The new files use `""` for missing values in `TOTAL_COLLECTED_ON_ACCOUNT`, `PAYMENT_PLAN_STATE`, `RESOLUTION_STATUS`, `HAS_EMAIL`, `HAS_PHONE`, `BALANCE_BAND`. The cell renderer only checks `value == null`, so empty strings pass through and cause formatting issues.

### DATA-03: No schema validation on cached data
In static mode (no Snowflake creds), cached JSON is served without schema validation. The schema validator only runs when querying Snowflake directly, so bad cached files fail silently downstream.

**Fix options:**
- Re-export the account files from Snowflake with `ACCOUNT_PUBLIC_ID` included and proper NULLs
- Or update `account-definitions.ts` to handle the missing column gracefully
- Add cached data schema validation in `fallback.ts`
