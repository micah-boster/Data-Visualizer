# Requirements: v5.0 External Intelligence

**Defined:** 2026-04-16
**Core Value:** Triangulate internal performance data with partner-reported scorecards and contractual targets so every partner conversation is grounded in the full picture.

## Scorecard Ingestion

- [ ] **SC-01**: User can upload a PDF scorecard and Claude extracts key metrics with field labels
- [ ] **SC-02**: User can upload an Excel/CSV scorecard and the app parses structured columns automatically
- [ ] **SC-03**: User can paste email body text and Claude extracts embedded metrics
- [ ] **SC-04**: First scorecard for a partner requires manual field mapping (partner metric names → internal vocabulary)
- [ ] **SC-05**: Subsequent scorecards from the same partner auto-map using the learned schema
- [ ] **SC-06**: Every extracted value shows a confidence indicator; low-confidence values are flagged for human review
- [ ] **SC-07**: User confirms/corrects every extraction before it's stored (human-in-the-loop)
- [ ] **SC-08**: Scorecard data is stored with partner ID, date, source format, and extraction metadata
- [ ] **SC-09**: Scorecard library shows all historical scorecards per partner, searchable by date

## Contractual Targets

- [ ] **TGT-01**: User can manually enter targets per partner per metric with effective date ranges
- [ ] **TGT-02**: User can upload a contract PDF and Claude extracts target clauses for confirmation
- [ ] **TGT-03**: Targets are typed: hard SLA (contractual), soft target (internal goal), or industry benchmark
- [ ] **TGT-04**: Targets are versioned — contract renewals archive old targets, don't overwrite
- [ ] **TGT-05**: Target data is accessible to charts, tables, and Claude NLQ context
- [ ] **TGT-06**: Editing a target shows the change history (who, what, when)

## Triangulation Views

- [ ] **TRI-01**: Partner detail view shows triangulation panel: internal metric, scorecard metric, and target side-by-side
- [ ] **TRI-02**: Divergence highlighting for metrics where internal and scorecard values disagree beyond threshold
- [ ] **TRI-03**: Target traffic-light per metric: green (on/above), amber (at risk), red (below)
- [ ] **TRI-04**: Overall partner health indicator aggregating target status across all metrics
- [ ] **TRI-05**: Trend view showing internal vs. scorecard vs. target over time
- [ ] **TRI-06**: Graceful display when scorecard or target data is unavailable for a metric
- [ ] **TRI-07**: Collection curve chart overlays target projection line alongside internal curves
- [ ] **TRI-08**: Claude NLQ answers questions referencing all three data sources

## Scorecard Reconciliation

- [ ] **REC-01**: Reconciliation dashboard per partner: timeline of scorecard submissions vs. internal snapshots
- [ ] **REC-02**: Drift detection: flag when scorecard-vs-internal gap changes significantly
- [ ] **REC-03**: Partner reliability indicator based on historical scorecard accuracy
- [ ] **REC-04**: Per-metric alignment history (some metrics may agree while others diverge)
- [ ] **REC-05**: Reconciliation summary exportable as data package for partner conversations

## Dynamic Curve Re-Projection

- [ ] **PROJ-01**: Projected curves re-calculate as actual collection data arrives
- [ ] **PROJ-02**: Three projection overlays: "at current pace", "to hit target", "partner-reported trajectory"
- [ ] **PROJ-03**: Confidence bands showing projection uncertainty
- [ ] **PROJ-04**: Projections respect anomaly detection — flagged batches show adjusted projections

## Future Requirements (deferred to v5.5+)

- **SC-10**: Auto-ingest scorecards when extraction confidence is high (graduate from human-in-the-loop)
- **SC-11**: Email integration: automatically detect and extract scorecards from partner emails
- **SC-12**: Scorecard format change detection: alert when a partner's scorecard format changes
- **TGT-07**: Industry benchmark data ingestion from public or purchased sources
- **TRI-09**: Cross-partner scorecard comparison (how do different partners report similarly-named metrics?)
