- [x] **DS-01**: Spacing scale defined as CSS custom properties on a 4px grid (space-1 through space-12) — shipped 26-01 as `--spacing-1..12` (namespace-driven rename) + semantic aliases (inline/stack/section/card-padding/page-gutter)
- [x] **DS-02**: Typography scale defined with named levels (display, heading, title, label, body, caption) each specifying font-size, line-height, weight, and letter-spacing — shipped 26-01 via `--text-*` with per-token `--line-height` / `--font-weight` / `--letter-spacing` sub-tokens
- [x] **DS-03**: Elevation system defined as multi-layer shadows (shadow-xs through shadow-lg) with light and dark variants — shipped 26-01 with warm-tinted light-mode layered shadows + darker tinted `.dark` overrides
- [x] **DS-04**: Motion tokens defined for duration (quick 100ms, normal 200ms, slow 350ms) and easing (default, spring, decelerate) — shipped 26-01 as `--duration-quick/normal/slow` + `--ease-default/spring/decelerate` (namespace-driven rename)
- [x] **DS-05**: Surface system defined with 5 named levels (base, raised, inset, overlay, floating) specifying background, border, and shadow treatment — shipped 26-01 with shadcn re-map (background/card/popover) preserving existing utilities
- [x] **DS-06**: All design tokens function correctly in both light and dark mode — shipped 26-01; user-verified light + dark in own browser; dark-mode surface-raised intentionally lighter than base per CONTEXT lock

## Out of Scope

| Feature | Reason |
|---------|--------|
| New API routes for chart data | Charting operates on client-side dataset |
| Chart data export (PNG/SVG) | Nice-to-have, not core workflow |
| Real-time data streaming | Batch refresh sufficient for 2-3 users |
| ML-based chart recommendations | Overkill for small dataset |
| Mobile-optimized UI | Desktop-first for internal users |
| User authentication | Small team, defer to v5 |
| Framer Motion or heavy animation lib | CSS transitions + Web Animations API sufficient |
