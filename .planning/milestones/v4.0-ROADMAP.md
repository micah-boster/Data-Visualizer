# Milestone v4.0: Design System & Daily-Driver UX

**Status:** NOT STARTED
**Phases:** 25-37
**Requirements:** See [v4.0-REQUIREMENTS.md](v4.0-REQUIREMENTS.md) (67 requirements)
**Absorbs:** v3.5 (Partner Lists, Chart Builder, Metabase Import) — resequenced after design foundation

## Overview

v4 is the "make it feel like a real product" release. The app has strong features (v1-v3.1) but they were built with ad-hoc styling — three different surface treatments, no spacing system, no elevation, no motion. Every future feature inherits that inconsistency.

v4 invests in the design foundation first, then builds new features on top of it. The order: code health fixes, design token system, visual polish, then the deferred features (Partner Lists, Charts, Metabase Import) land polished from day one.

**Success criteria:** Someone outside the partnerships team sees it and says "this looks like a real product." The daily experience of using the tool feels fast, quiet, and confident.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Absorb v3.5 into v4 | v3.5 hadn't started. Building features on ad-hoc styling creates retrofit debt. Design first, features inherit. |
| Code health before design | Bug fixes and React anti-patterns should be cleaned before touching visual layer |
| Tokens before surfaces | Can't build consistent surfaces without a spacing/shadow/type vocabulary |
| Phases 27 + 28 can run in parallel | Typography changes text classes, surfaces change container classes — low conflict |
| Phases 30 + 34 can run in parallel | Motion is CSS polish on existing components, Partner Lists is new feature code — different files |
| Phases 36 + 37 can run in parallel | Chart Builder is rendering/UI, Metabase Import is parsing/mapping — fully independent |
| URL nav as independent track | Structural change that doesn't depend on design work, can run anytime after 25 |
| Merge Chart Renderer + Builder into Phase 36 | Tighter feedback loop: build renderer and UI together after design system exists |
| Accessibility after polish | Audit the final visual state, not an intermediate one |
