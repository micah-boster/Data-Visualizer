# Requirements: v4.5 Correctness & Foundation

**Defined:** 2026-04-21 ‚Äî Expanded 2026-04-26 with multi-lens audit findings (data correctness review, structural/systems review, design-thinking review, code-quality review).
**Core Value:** Structural guardrails before v5.0 ‚Äî verified metrics (so triangulation doesn't amplify silent bugs into partner-relationship damage), a documented threat model for the ingestion surface (so Phase 45 architects security in, not retrofits it), typed boundaries (so v5.0 doesn't inherit and amplify representation debt √ó 3 new data sources), and locked vocabulary (so v5.0's five new domain terms layer cleanly on a defined model).

## Theme

v4.5 holds only the work whose value degrades if deferred past v5.0. Four lenses ran across the codebase in late April 2026 and surfaced four distinct categories of work, each with a different degradation curve:

- **Data correctness** (degrades fast) ‚Äî silent bugs in metrics will compound when triangulation amplifies them. ‚Üí Phase 41.
- **Security architecture** (degrades fast) ‚Äî retrofitting threat models into shipped ingestion is expensive. ‚Üí Phase 42.
- **Boundary hardening** (degrades fast) ‚Äî v5.0 adds 3 new data shapes (scorecards, targets, triangulation rows). If they each get parsed ad-hoc the way batch rows are now, v5.0 ships with `Record<string, unknown>` debt √ó 3. ‚Üí Phase 43.
- **Vocabulary lock** (degrades fast) ‚Äî v5.0 adds 5 new domain terms (scorecard, target, triangulation, reconciliation, divergence) on top of the existing 12. Without a glossary and term registry first, the conceptual sprawl compounds. ‚Üí Phase 44.

Behavioral QA, component decomposition, state consolidation, and test pyramid inversion are deferred to v5.5 ‚Äî those audits improve when informed by real post-v5.0 usage. See "Deferred to v5.5" section below.

If "your numbers" have silent bugs, triangulation surfaces them as "the partner disagrees with us" ‚Äî eroding trust in both the tool and the partner relationship. Every metric on screen must match what a direct Snowflake query returns, at every scope level, before external data lands.

## Boundary Hardening (Phase 43)

**Principle:** v5.0 introduces three new data shapes ‚Äî scorecard rows, target rows, triangulation rows ‚Äî each parsed at their own boundary. If those parsers replicate the current `Record<string, unknown>` pattern that batch rows live in, v5.0 inherits 3√ó the representation debt. Phase 43 establishes the boundary-discipline pattern (parse once, type the interior, version persistence, unify chart primitives) on existing surfaces so v5.0 inherits a typed substrate, not a string-keyed one.

- [x] **BND-01**: Canonical row representation at the parser boundary. `BatchRow` and `AccountRow` interfaces with resolved types ‚Äî numbers as `number | null` (rate-shaped fields can be null per DCR-08), branded `BatchAgeMonths = number & { __brand: 'months' }`, long-format `curve: CurvePoint[]` baked in. Single parser at `src/lib/data/parse-batch-row.ts`. Static cache JSON loads through the same parser, eliminating the days-vs-months ambiguity by construction. _(Phase 43-01, 2026-04-30 ‚Äî `src/lib/data/types.ts` exports BatchRow/AccountRow/BatchAgeMonths/CurvePoint/asBatchAgeMonths/ParseBatchRowsResult; `parseBatchRow` returns `BatchRow | null` with branded age + baked long-format curve + filter-and-warn drops; `parseBatchRows` partitions input into typed rows + drop metadata; static cache fallback routes bundled JSON through the parser at boot)_
- [x] **BND-02**: Compute layer signature change. Every function in `src/lib/computation/` takes `BatchRow[]`, not `Record<string, unknown>[]`. The bag-of-strings does not appear in `src/lib/computation/`. Three duplicate `coerceAgeMonths` functions collapse to one (the parser). _(Phase 43-01, 2026-04-30 ‚Äî 8 compute files migrated: compute-kpis, compute-norms, compute-anomalies, compute-trending, compute-cross-partner, reshape-curves, metric-eligibility, segment-split; three duplicate `coerceAgeMonths` collapsed to one `asBatchAgeMonths` in `types.ts`; use-partner-stats parses once and surfaces non-blocking sonner toast on drops; all 13 smokes pass green; static-cache parity baseline preserved exactly)_
- [x] **BND-03**: Versioned, safe persistence layer. Every localStorage blob carries `_meta: { schemaVersion: N, savedAt: ts }`. Migration chain on load (missing migrator = loud failure, not silent drop). Verified writes (read-back-and-compare; surface error if quota or storage failure). Cross-tab `storage` event listener ‚Äî in-memory state syncs when another tab writes. Replace direct localStorage in `views/storage.ts`, `columns/persistence.ts`, partner-lists, chart-presets. _(Phase 43-02a, 2026-05-01 ‚Äî `createVersionedStore<T>` at `src/lib/persistence/versioned-storage.ts` with envelope + migration chain + verified write + cross-tab subscribe; `MissingMigratorError` throws in dev / drops + console.error in prod; all 5 storage modules (views, columns, partner-lists, partner-config, chart-presets) wrapped at schemaVersion 1 with empty migration maps; zero direct `localStorage.setItem/getItem` in those modules; smoke covers all 6 cases)_
- [x] **BND-04**: Snowflake reliability primitives. Retry with exponential backoff on documented transient error codes. Queue-wait visibility separated from query-execution time. Request-id correlation (uuid logged server-side, returned in `X-Request-Id` response header). Sanitized client errors (strip Snowflake-internal table/column names). Lightweight circuit-breaker (5 consecutive failures ‚Üí 60s degraded mode using stale React Query cache). _(Phase 43-02b, 2026-05-01 ‚Äî `executeWithReliability` at `src/lib/snowflake/reliability.ts` wraps all 3 Snowflake-touching API routes (`/api/data`, `/api/accounts`, `/api/curves-results`) with retry (1s/2s backoff, 3 total attempts) + circuit breaker (5 consecutive failures ‚Üí 60s open) + `Server-Timing` queue/execute split + structured logging; `isTransientSnowflakeError` classifies network codes (ECONNRESET/ETIMEDOUT/ENOTFOUND/ECONNREFUSED/EAI_AGAIN) and Snowflake codes (390114/000625/000604) as retriable; sanitized client errors with `X-Request-Id` correlation header; `<DegradedBanner>` mounted in app/layout.tsx surfaces stale-data state; `(stale)` badge near last-updated timestamp; static-cache fallback path unaffected)_
- [x] **BND-05**: ChartFrame primitive ‚Äî single `<ChartFrame>` component owning title, subtitle, legend slot, tooltip slot, empty/loading/error state, stale-column warning, density-aware padding, metric-polarity prop (DCR-09). ## Ingestion-Surface Security Review (Phase 42)

**Principle:** v5.0 Phase 45 introduces a large new security surface ‚Äî file uploads, Claude extraction, stored scorecard data, potentially PII in contracts. Architecting security in is cheap; retrofitting is expensive and leaves residual risk. v4.5 audits what exists today AND produces the threat model Phase 45 needs.

**Phase 42 split (2026-04-30):** OAuth on Vercel is not yet configured. Roughly 70% of Phase 42's value is OAuth-independent ‚Äî including the load-bearing forward threat model that Phase 45 architecture depends on. Splitting Phase 42 into 42a (do now, OAuth-independent) and 42b (deferred, OAuth-dependent) lets v4.5 close cleanly while preserving the load-bearing piece. SEC-04 specifically must NOT slip past the start of Phase 45.
