**Edit B — `src/hooks/use-saved-views.ts` sanitizer:**

Modify `sanitizeSnapshot` to accept a `knownListIds: Set<string>` parameter and strip `listId` when it references a missing list:

```typescript
function sanitizeSnapshot(snapshot: ViewSnapshot, knownListIds: Set<string>): ViewSnapshot {
  return {
    ...snapshot,
    // …existing column-key sanitization…
    listId:
      snapshot.listId && knownListIds.has(snapshot.listId)
        ? snapshot.listId
        : undefined,  // strip when unknown — non-destructive at load time
  };
}

function sanitizeViews(views: SavedView[], knownListIds: Set<string>): SavedView[] {
  return views.map((v) => ({ ...v, snapshot: sanitizeSnapshot(v.snapshot, knownListIds) }));
}
```

Inside `useSavedViews`, accept the known list IDs. The cleanest wiring is to accept them as an optional parameter from the caller:

```typescript
export function useSavedViews(knownListIds?: Set<string>) {
  // …existing…
  useEffect(() => {
    let loaded = loadSavedViews();
    if (loaded.length === 0) loaded = getDefaultViews();
    const ids = knownListIds ?? new Set<string>();
    setViews(sanitizeViews(loaded, ids));
    hasHydrated.current = true;
  }, [knownListIds]);
```

Callers that don't need list-awareness (if any still exist) pass nothing and get an empty set (effectively stripping all listIds on load — safe conservative default). Callers that DO care pass `new Set(lists.map((l) => l.id))` from `usePartnerLists()`.
