**File 1: `src/lib/computation/compute-trending.ts`**

Current shape (per RESEARCH): returns a single boolean `insufficientHistory: true` when `rows.length < 3`. KPI-04 wants per-horizon suppression (3mo card needs ≥3 prior batches that reached 3mo; 6mo card needs ≥3 prior that reached 6mo; 12mo card needs ≥3 prior that reached 12mo; "since inception" shows delta with any prior batch).

Strategy: extend the returned trend object with per-horizon suppression booleans:

```ts
export interface TrendResult {
  // ...existing fields (delta, rollingAvg, insufficientHistory for legacy callers)
  suppressDelta: {
    rate3mo: boolean;
    rate6mo: boolean;
    rate12mo: boolean;
    rateSinceInception: boolean;
  };
}
```

Computation:
1. Sort `rows` by batch age descending (latest batch first).
2. Let `priorBatches = rows.slice(1)` (exclude the latest — "prior" per CONTEXT lock and compute-trending.ts:40 precedent).
3. For each horizon H ∈ {3, 6, 12}:
   - Count `priorBatches.filter((r) => ageInMonths(r) >= H).length`.
   - `suppressDelta[rateXmo] = count < 3`.
4. `suppressDelta.rateSinceInception = priorBatches.length === 0`.

Use the same age-coercion helper as Task 1 (and Plan 05). If the trending function currently doesn't take raw rows, thread the age source through; else adapt to the existing per-batch records.

Leave `insufficientHistory` on the result (don't break legacy callers — tag it as "deprecated: use suppressDelta.{horizon} instead" in a JSDoc line).

**File 2: `src/lib/computation/suppression.smoke.ts` (NEW)**

```ts
import assert from 'node:assert/strict';
// Import the pure helper that computes suppression — if compute-trending.ts
// bundles it with side effects, extract it into a named export (e.g., `computeSuppression`).

import { computeSuppression } from './compute-trending.ts';

// Helper to build synthetic batches.
const batch = (age: number) => ({ ageInMonths: age });

// Scenario 1: 4 batches all at age 12mo.
// Prior batches = 3 (exclude latest) — all reached every horizon ≥ delta shown at all horizons.
const s1 = computeSuppression([batch(12), batch(12), batch(12), batch(12)]);
assert.equal(s1.rate3mo, false);
assert.equal(s1.rate6mo, false);
assert.equal(s1.rate12mo, false);
assert.equal(s1.rateSinceInception, false);

// Scenario 2: 4 batches, ages 2, 4, 5, 7 — prior = 3 batches at 2/4/5mo.
// 3mo horizon: 2 prior reached (4+5) < 3 → suppress.
// 6mo horizon: 0 prior reached → suppress.
// 12mo horizon: 0 prior reached → suppress.
// Latest batch (7mo) is excluded from prior count.
const s2 = computeSuppression([batch(7), batch(2), batch(4), batch(5)]);
assert.equal(s2.rate3mo, true, 'only 2 prior reached 3mo');
assert.equal(s2.rate6mo, true);
assert.equal(s2.rate12mo, true);
assert.equal(s2.rateSinceInception, false, 'prior exists');

// Scenario 3: 1 batch total → no priors.
const s3 = computeSuppression([batch(15)]);
assert.equal(s3.rate3mo, true);
assert.equal(s3.rateSinceInception, true, 'no priors → suppress even since-inception');

console.log('suppression smoke OK');
```
