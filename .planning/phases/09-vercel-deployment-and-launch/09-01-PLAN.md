1. Update `next.config.ts` to add `snowflake-sdk` to `serverExternalPackages`. This prevents Next.js from bundling the heavy snowflake-sdk dependency tree (AWS SDK, Azure, Google Cloud) into serverless functions, which would cause MODULE_NOT_FOUND errors or exceed size limits at runtime.

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  serverExternalPackages: ['snowflake-sdk'],
};

export default nextConfig;
```

2. Create `src/app/api/health/route.ts` -- a lightweight GET endpoint that confirms the app is running and Snowflake is reachable. Import `executeQuery` from `@/lib/snowflake/queries`. Return JSON with:
   - `status`: "ok" or "degraded"
   - `timestamp`: ISO string
   - `version`: first 7 chars of `process.env.VERCEL_GIT_COMMIT_SHA` or "local"
   - `snowflake.status`: "connected" or "error"
   - `snowflake.latencyMs`: milliseconds for a `SELECT 1 AS ping` query
   - On Snowflake error: set status to "degraded", include error message, return HTTP 503

Mark the route as `export const dynamic = 'force-dynamic'` to prevent caching.

Reference the research pattern in 09-RESEARCH.md Pattern 2 for exact implementation shape.
