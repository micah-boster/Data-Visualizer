  <name>Task 1: Create data pivot utility and partner average computation</name>
  <files>src/components/charts/pivot-curve-data.ts</files>
  <action>
Create `src/components/charts/pivot-curve-data.ts` with these exports:

1. **`BATCH_KEY_PREFIX`** constant (`"batch_"`) -- used to sanitize batch names into safe Recharts dataKeys. Batch names from Snowflake may contain dots/brackets that break Recharts property access.

2. **`PivotedPoint` type:**
```typescript
interface PivotedPoint {
  month: number;
  [key: string]: number | undefined;
}
```

3. **`BatchKeyMap` type** -- maps sanitized keys back to display names:
```typescript
type BatchKeyMap = Map<string, string>; // batch_0 -> "2024-Q1 Batch"
```

4. **`pivotCurveData(curves: BatchCurve[], metric: 'recoveryRate' | 'amount')`** returns `{ data: PivotedPoint[], keyMap: BatchKeyMap }`:
   - Collect all unique months across all curves into a Set, sort ascending
   - For each month, create a PivotedPoint with `month` and one key per batch
   - Map batch names to safe keys: `batch_0`, `batch_1`, etc. (index based on input order)
   - CRITICAL: Use `undefined` (NOT 0) for months where a batch has no data point (beyond its age). This prevents false zero cliffs -- Recharts skips undefined values.
   - Return both the pivoted data array and the key map for tooltip/legend display

5. **`addAverageSeries(data: PivotedPoint[], curves: BatchCurve[], metric: 'recoveryRate' | 'amount')`** returns `PivotedPoint[]`:
   - For each pivoted point, compute mean of all batch values at that month (excluding undefined)
   - Add as `__avg__` key to each point
   - If no batches have data at a month, set `__avg__` to undefined
   - Return new array (don't mutate input)

Import `BatchCurve` from `@/types/partner-stats`. Do NOT import from reshape-curves (no dependency needed -- pivot operates on the BatchCurve output).

Include JSDoc comments on each export explaining purpose and the "why" behind safe key mapping.
  </action>

  </verify>
  <done>
    - pivotCurveData transforms BatchCurve[] into Recharts-compatible flat array
    - Missing months use undefined, not 0
    - Batch names sanitized to safe keys with lookup map
    - addAverageSeries computes mean __avg__ per month
    - TypeScript compiles without errors
  </done>

<task type="auto">
  <name>Task 2: Create chart interaction state hook</name>
  <files>src/components/charts/use-curve-chart-state.ts</files>
  <action>
Create `src/components/charts/use-curve-chart-state.ts` exporting a `useCurveChartState` hook.

The hook accepts `curves: BatchCurve[]` and returns all interactive state + handlers:

**State:**
- `metric`: `'recoveryRate' | 'amount'` -- default `'recoveryRate'` (CURVE-02 requirement)
- `soloedBatch`: `string | null` -- the sanitized key of the currently soloed batch, or null
- `hiddenBatches`: `Set<string>` -- sanitized keys of batches hidden via legend toggle
- `showAverage`: `boolean` -- default `true` (partner avg reference line, CURVE-06)
- `showAllBatches`: `boolean` -- default `false` (cap at 8 most recent)

**Derived (via useMemo):**
- `sortedCurves`: curves sorted by `ageInMonths` ascending (newest = lowest age first)
- `newestBatchKey`: the sanitized key of the most recent batch (index 0 after sort)
- `visibleBatchKeys`: array of sanitized keys that should render. When `showAllBatches` is false, take the first 8 from sortedCurves. When true, take all. Filter out `hiddenBatches`.
- `defaultVisibleKeys`: the first 8 (or all if fewer) sanitized keys before hidden filter -- used by legend to show which are "default visible"

**Handlers:**
- `toggleMetric()`: swap between recoveryRate and amount
- `handleLineClick(batchKey: string)`: toggle solo mode (click same = unsolo, click different = solo that one)
- `toggleBatchVisibility(batchKey: string)`: add/remove from hiddenBatches set
- `toggleAverage()`: flip showAverage
- `toggleShowAll()`: flip showAllBatches

**Opacity helper:**
- `getLineOpacity(batchKey: string)`: returns number
  - If soloed: soloed batch = 1, others = 0.08
  - If not soloed: newest batch = 1, others = 0.5

**Stroke width helper:**
- `getLineStrokeWidth(batchKey: string)`: newest = 3, others = 1.5 (per user decision)

Import `BatchCurve` from `@/types/partner-stats`. Use `useState` and `useMemo` from React.
  </action>

    - Derived visibleBatchKeys respects 8-batch cap and hidden set
    - Solo mode, legend toggle, metric toggle, average toggle all have handlers
    - Opacity and stroke width helpers implement user's visual hierarchy decisions
    - TypeScript compiles without errors
  </done>
