**Step 1 — Identify the drill boundary.** Open data-display.tsx. Locate the region starting around line 494 where `drillState.level` gates rendering. This includes:

The boundary is whatever renders DIFFERENTLY across drill levels. Header (`<Header />`), sidebar (`<AppSidebar />`), and any sticky chrome must NOT be inside the cross-fade wrapper — they stay mounted.

**Step 2 — Apply re-key + transition-opacity (primary approach).** Wrap the drill-variant region in a wrapper that:
- Keys on drill identity: `key={`drill-${drillState.level}-${drillState.partner ?? 'none'}-${drillState.batch ?? 'none'}`}`
- Applies `className="transition-opacity duration-normal ease-default"`
- Uses a `data-drill-fade` attribute for guard visibility and potential future styling hooks

Pattern:

```tsx
<div
  key={`drill-${drillState.level}-${drillState.partner ?? 'none'}-${drillState.batch ?? 'none'}`}
  data-drill-fade
  className="transition-opacity duration-normal ease-default"
>
  {/* KPI strip + charts + table — the existing drill-variant render flow */}
  {drillState.level === 'partner' && <KpiSummaryCards ... />}
  <SectionErrorBoundary resetKeys={[data]}>
    {/* charts region */}
  </SectionErrorBoundary>
  {/* table region */}
</div>
```

Re-keying causes React to unmount the old subtree and mount the new one. CSS `transition-opacity` applies to the NEW mount because React assigns the utility class on first render — the browser interpolates from the initial un-styled paint (effectively opacity: 1 from the start since there's no starting-opacity animation here; this is a re-key fade, not a true cross-fade). This matches Notion-restrained personality (subtle, not theatrical).

**Step 3 — Guard Pitfall 3 (scroll jump).** Drill router.push already uses `{ scroll: false }` (verified use-drill-down.ts:29-32). Add a CSS `contain: layout` hint on the wrapper to hold the scroll container height stable mid-fade:

```tsx
style={{ contain: 'layout' }}
```

(Inline style is acceptable here — it's a layout-containment hint, NOT a motion timing value, so check:motion doesn't flag it. check:motion regex matches only `transitionDuration|transitionTimingFunction`, not `contain`.)

**Step 4 — Decide re-key vs dual-mount.** Ship with the re-key approach (Step 2). During manual verification (see <verification>), test the partner-with-many-batches → batch-detail drill. If the table shows a visible flash of empty state mid-fade:

UPGRADE PATH (only if flash is visible):

- Requires React state tracking previous drillState for ~250ms (duration-normal + a small tail buffer).
- Implementation: `usePreviousDrill` state + two layered divs with opacity-0/opacity-100 classes.

Do NOT upgrade speculatively — re-key is simpler, handles URL back/forward correctly via React's natural re-render, and matches CONTEXT "keep it simple." Document the escalation decision in SUMMARY.

**Step 5 — Verify header/sidebar untouched.** Grep the file for `<Header` and `<AppSidebar` — confirm neither is inside the new wrapper. If either IS inside the existing drill region (unlikely per RESEARCH but verify), split them out above the wrapper.
