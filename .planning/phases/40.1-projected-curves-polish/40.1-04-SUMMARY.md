## Gap Closures

### Gap 1 — `footer-aggregate-unit-mismatch` (CLOSED)

**Reference:** `40.1-VERIFICATION.md` frontmatter `gaps[0]`.

**Root cause (from VERIFICATION.md):** `buildModeledColumns()` in `src/lib/columns/definitions.ts` tags both modeled and delta cols with `meta: { type: 'percentage' }`. Cell renderers compensate for the 0..100 scale (modeled cell does `(value as number) / 100`; delta cell uses `ModeledDeltaCell` independently). But `TableFooter` calls `formatAggregate(agg.primary, 'percentage', 'Avg')` → `formatPercentage(rawAvg)` which multiplies by 100. Net display error: modeled cells render `~3000-5000%` when they should be `~30-50%`; delta cells render `%` suffix when their body cells display `pp`.

**Fix:** Per-column `meta.footerFormatter` escape hatch (Path A from plan).

- **`src/lib/columns/definitions.ts`** — extended each of the four modeled+delta col-template `meta` objects with a `footerFormatter` callback:
  - **Modeled col template:** `(avg, label) => avg == null ? '\u2014' : ${label}: ${avg.toFixed(1)}%` — direct `.toFixed(1)` on the 0..100 average, no `/100` divide.
  - **Delta col template:** `(avg, label) => avg == null ? '\u2014' : ${label}: ${sign}${avg.toFixed(1)}pp` — signed, `pp` suffix, matches body-cell unit convention from `ModeledDeltaCell`.
- **`src/components/table/table-footer.tsx`** — extended the `meta` type cast to include optional `footerFormatter`; consult it before the generic `formatAggregate(agg.primary, colType, agg.label)` fallback. Pure addition: any column without `footerFormatter` behaves exactly as before.

**Why Path A over Path B (per-column override over new meta.type values):** Path A is surgical, type-safe, narrow blast radius. Path B would have required new `meta.type` values like `'percentage-100'` / `'percentage-points'`, which other consumers (cell renderers, aggregations, formatters) might also need to interpret — expanding scope for one cell type's footer. VERIFICATION.md fix_options also tracked Path A as the recommendation.

**Anti-fragility — what was deliberately NOT done:**
- `formatPercentage` in `numbers.ts` UNTOUCHED (30+ callers expect 0..1 scale).
- `aggregations.ts` UNTOUCHED (the `agg.primary` numeric is correct on 0..100; only the formatter was wrong).
- `meta.type` for the modeled+delta cols stays `'percentage'` (cell renderer + formatPercentage callers use it; pivoting to a new type would cascade).

### Gap 2 — `header-truncation-illegible` (CLOSED)

**Reference:** `40.1-VERIFICATION.md` frontmatter `gaps[1]`.

**Root cause (from VERIFICATION.md):** `src/components/table/draggable-header.tsx:110` applied `className="truncate min-w-0 max-w-[180px]"` (POL-06 from Phase 38 — single-line truncate with native `title` tooltip on overflow). User feedback 2026-04-26: "impossible to use if you can't read any column titles without a tooltip". The truncate-with-tooltip pattern was broadly hostile, not specific to the new modeled+delta cols.

**Fix:** Global swap of label span classes — POL-06 reversal.

- **`src/components/table/draggable-header.tsx`** — swap header label `<span>` className from `truncate min-w-0 max-w-[180px]` → `line-clamp-2 break-words leading-snug min-w-0`.
  - **`line-clamp-2`** — wraps to up to 2 lines, then ellipsis (Tailwind utility; uses `-webkit-line-clamp` under the hood, supported in all evergreen browsers).
  - **`break-words`** — allows long unbreakable strings to break mid-word as a last resort.
  - **`leading-snug`** — tightens line-height to 1.375 so two lines feel cohesive.
  - **`min-w-0`** — preserved from original; required for the flex parent's wrap mechanics to honor the column's `size`.
  - **`max-w-[180px]` removed** — line-clamp-2 honors the column's actual width (set per-col via `ColumnDef.size`); imposing a 180px cap would constrain longer columns. Per-column size is the right constraint.
- **Comment block above the span updated** — replaces the original POL-06 truncate comment with a reversal comment that records: cross-reference to Phase 40.1 Plan 04 Gap 2, user-feedback date (2026-04-26), feedback verbatim, why max-w cap was dropped, and that native `title` is retained as fallback (with the original POL-06 rationale for preferring `title` over Tooltip primitive carried forward).
- **Native `title` attribute retained** — last-resort fallback for the rare 3+ line edge case. Original POL-06 rationale (header re-renders on drill/filter change make per-cell Tooltip mounts costly; drag pointer events conflict with hover-open Tooltips) still applies for the fallback path.

**Verification observations:**
- `<th>` chrome still has `overflow-hidden` (line 76); harmless for line-clamp-2 because the th has no fixed height — it grows naturally to fit 2 lines, and line-clamp-2 itself handles the ellipsis truncation. `overflow-hidden` only matters for horizontal escape, which `break-words` prevents.
- `<thead>` in `table-header.tsx` has `sticky top-0` with no fixed height on thead/tr/th — the row will simply grow taller and stay pinned. NO edits needed in `table-header.tsx`.
- `<div className="flex cursor-pointer items-center..." />` parent (line 80) keeps the label vertically centered against the SortIndicator and ColumnHeaderFilter siblings — when label grows to 2 lines, indicators stay centered against the label's vertical midpoint.
- Resize handle (line 133, `top-0 h-full`) stretches to whatever the now-potentially-taller th's height is.

**Anti-fragility — what was deliberately NOT done:**
- No `min-h` or `h` added on `<th>`, `<thead>`, or `<tr>`. Header should grow naturally; forcing min-height creates dead space when no labels need 2 lines.
- No per-column `wrap` opt-in on ColumnDef. Globally beneficial; per-column adds API surface for no gain.
- `title` attribute NOT removed — fallback for the rare 3+ line case.
- `table-header.tsx` UNTOUCHED — sticky + bg + shadow chrome correct as-is.
