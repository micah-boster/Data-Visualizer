```typescript
interface TrendIndicatorProps {
  trend: BatchTrend;
  formattedValue: string;    // The already-formatted metric value (e.g. "78.2%")
  columnType: string;        // Column type for formatting rollingAvg in tooltip
  lowConfidence: boolean;    // true when baseline uses < 4 batches
}
```

Component logic:
1. Determine arrow character: direction 'up' → '↑', 'down' → '↓', 'flat' → '—'
2. Determine color using polarity:
   - If direction is 'flat': use `text-muted-foreground` (gray)
   - If direction is 'up' AND polarity is 'higher_is_better': `text-emerald-600 dark:text-emerald-400` (green = good)
   - If direction is 'up' AND polarity is 'lower_is_better': `text-red-500 dark:text-red-400` (red = bad)
   - If direction is 'down' AND polarity is 'higher_is_better': `text-red-500 dark:text-red-400` (red = bad)
   - If direction is 'down' AND polarity is 'lower_is_better': `text-emerald-600 dark:text-emerald-400` (green = good)
3. If lowConfidence: add `opacity-50` class to the arrow span
4. Build tooltip text:
   - Format the rolling average using getFormatter(columnType)
   - Direction label: 'Up' / 'Down' / 'Flat'
   - Text: `"{Direction} {Math.abs(deltaPercent).toFixed(1)}% vs {baselineCount}-batch avg ({formattedRollingAvg})"`
   - Example: "Up 12.3% vs 4-batch avg (69.5%)"
   - For flat: "Flat vs 4-batch avg (69.5%)" (skip the delta percentage)
5. Render pattern:
   ```
   <Tooltip>
     <TooltipTrigger asChild>
       <span className="inline-flex items-center gap-1">
         {formattedValue}
         <span className={`text-xs font-medium ${colorClass} ${opacityClass}`}>{arrow}</span>
       </span>
     </TooltipTrigger>
     <TooltipContent>{tooltipText}</TooltipContent>
   </Tooltip>
   ```
