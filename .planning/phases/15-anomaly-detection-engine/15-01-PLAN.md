<task type="auto">
  <name>Task 2: Core anomaly computation module</name>
  <files>src/lib/computation/compute-anomalies.ts</files>
  <action>
Create `src/lib/computation/compute-anomalies.ts` following the pattern of `compute-trending.ts` (pure function, typed input/output, const metric lists).

**Exports:**

1. `ANOMALY_METRICS` -- readonly array of all ~11 curated metric column keys (5 performance + 6 diagnostic, per RESEARCH.md recommended list)

2. `METRIC_GROUPS` -- record mapping group keys to `{ label: string; metrics: string[] }`. Four groups per RESEARCH.md:
   - `funnel`: "Funnel Degradation" (penetration, conversion, plans)
   - `collection`: "Collection Underperformance" (lifetime, 6mo, 12mo, payments)
   - `portfolio_quality`: "Portfolio Quality Shift" (avg amount placed, credit score)
   - `engagement`: "Engagement Drop" (SMS open rate, email open rate)

3. `computeAnomalies(rows, norms, portfolioNorms?)` -> `AnomalyReport`
   - Takes a single partner's rows, their norms (or portfolio norms for fallback), and optional portfolio norms
   - For each row (batch):
     a. Compute z-score for each metric in ANOMALY_METRICS: `(value - norm.mean) / norm.stddev`
     b. Skip metric if: norm missing, norm.count < 2, norm.stddev === 0, value is null/NaN
     c. Flag only if deviation is in the "bad" direction per `getPolarity()`:
        - `higher_is_better` AND zScore < -2 => flagged (below = bad)
        - `lower_is_better` AND zScore > 2 => flagged (above = bad)
     d. Batch is flagged when `flags.length >= 2`
     e. Group flags into `AnomalyGroup[]` using METRIC_GROUPS. For each group with 1+ matching flags, create an AnomalyGroup with avgDeviation = mean of abs(z-scores) in that group. Flags not matching any group remain ungrouped (create a group with key "other", label "Other").
     f. Compute severity: `flags.length * avgDeviation * Math.log(Math.max(totalAmountPlaced, 1))` where totalAmountPlaced comes from `row.TOTAL_AMOUNT_PLACED`. Guard log(0) with Math.max.
   - Sort batches by BATCH string (same as compute-trending.ts) to identify latest batch
   - Build PartnerAnomaly roll-up:
     - `isFlagged`: latest batch's isFlagged
     - `severityScore`: latest batch severity as primary (per RESEARCH.md recommendation)
     - `flaggedBatchCount`: count of flagged batches
     - `latestBatch`: the most recent BatchAnomaly
     - `batches`: all BatchAnomaly objects
     - `usedPortfolioFallback`: false (caller sets this)

4. `computeAllPartnerAnomalies(allRows)` -> `Map<string, PartnerAnomaly>`
   - Groups allRows by PARTNER_NAME
   - Computes portfolio norms once: `computeNorms(allRows)`
   - For each partner:
     - If partnerRows.length < 3: use portfolioNorms, set `usedPortfolioFallback = true`
     - If partnerRows.length >= 3: use `computeNorms(partnerRows)`, set `usedPortfolioFallback = false`
     - Call `computeAnomalies(partnerRows, norms, portfolioNorms)`
   - Returns Map keyed by partner name

**Edge cases to handle:**
- Zero stddev: skip metric (do not flag)
- Null/NaN values: skip metric
- Single-batch partners: use portfolio fallback
- `TOTAL_AMOUNT_PLACED` missing or 0: use `Math.log(1)` = 0 (severity becomes 0, acceptable)
- No flagged metrics on a batch: severityScore = 0, isFlagged = false, groups = []

**Import from existing modules:**
- `computeNorms` from `./compute-norms`
- `getPolarity` from `./metric-polarity`
- Types from `@/types/partner-stats`
