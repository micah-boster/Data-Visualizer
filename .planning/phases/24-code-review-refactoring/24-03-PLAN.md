    **Categories and known issues (minimum — add any new findings from Task 1):**

    **Data Layer:**
    - KI-01: Snowflake responses not Zod-validated for live queries (Medium) — only cached data has schema validation. Suggested fix: Add Zod schema to `/api/data` and `/api/accounts` response parsing.
    - KI-02: Dynamic column key strings not type-checked (Low) — `Record<string, unknown>` means typos in column key strings like `row['PARTNER_NAMEE']` are not caught. Suggested fix: Create a `ColumnKey` string literal union type from COLUMN_CONFIGS keys.
    - KI-03: No retry logic on Snowflake query failures (Medium) — API routes return error immediately on failure. Suggested fix: Add exponential backoff retry (1-2 attempts) in the API route handler.
    - KI-04: No request deduplication for imperative fetches (Low) — TanStack Query handles `useData` dedup but imperative `fetch` calls could overlap. Suggested fix: Unlikely to be triggered in practice; document as theoretical.

    **UI/UX:**
    - KI-05: Chart expansion state uses `localStorage` read in `useState` initializer (Low) — causes potential hydration mismatch (server defaults to `true`, client reads localStorage). Suggested fix: Read localStorage in a `useEffect` to avoid SSR mismatch.
    - KI-06: Schema warning dismissal is ephemeral (Low) — `useState` resets on page refresh. Suggested fix: Persist dismissal to localStorage or sessionStorage.
    - KI-07: Dimension filter at root level doesn't reduce table rows (Medium) — known bug from v2. Suggested fix: Investigate filter predicate logic in use-filter-state.ts for root-level dimension handling.

    **Architecture:**
    - KI-08: `data-display.tsx` is the largest non-UI-primitive component at ~450 LOC (Low) — handles orchestration + layout + state. Well-structured with clear sections but a candidate for future decomposition. Suggested fix: If it grows further, extract chart section and context wiring into sub-components.
    - KI-09: Three eslint-disable comments in `use-filter-state.ts` (Low) — for `react-hooks/exhaustive-deps` on `searchParams.toString()` string optimization. Suggested fix: Extract a `useStableSearchParams()` hook that returns a stable string representation.
    - KI-10: `lib/columns/` files have `'use client'` directive (Low) — unusual for a `lib/` directory but correct because column definitions reference client components via `createElement`. Suggested fix: No fix needed; document as intentional.
    - KI-11: `console.error`/`console.warn` in production code (Low) — 5 instances across API routes and schema validation. Suggested fix: Replace with a structured logger (e.g., pino) when observability becomes a priority.

    **Performance:**
    - KI-12: `computeKpis` runs for all partner groups on every `allData` change (Low) — called in `QuerySearchBarWithContext` useMemo. The memo prevents unnecessary recomputation, but the full partner-group aggregation runs whenever batch data changes. Suggested fix: Cache KPI results in a context or move to React Query.
    - KI-13: No error boundary wrapping individual sections (Medium) — a chart render error takes down the entire DataDisplay component. Suggested fix: Wrap chart sections and table in individual `<ErrorBoundary>` components.

    **Missing Features / Future Work:**
    - KI-14: No accessibility audit (Medium) — ARIA labels, keyboard navigation, and screen reader compatibility have not been tested. Suggested fix: Run axe-core audit, add ARIA labels to interactive elements.
    - KI-15: Dynamic curves don't re-project based on actuals (Low) — curves are static projections. Suggested fix: Recalculate projection curve endpoints using actual collection data as it arrives.
    - KI-16: Metabase query import not implemented (Low) — planned for Milestone 3.1. Accept/translate MBQL or SQL into the app's query system. Suggested fix: Build a query parser that maps Metabase question JSON to the app's column/filter model.
    - KI-17: No structured logging (Low) — uses `console.error`/`console.warn`. Suggested fix: Adopt pino or similar; route server logs through Vercel's log drain.
    - KI-18: Snowflake credentials not provisioned (Medium) — app runs on static cache only. Suggested fix: Add `SNOWFLAKE_ACCOUNT`, `SNOWFLAKE_USER`, `SNOWFLAKE_PASSWORD`, `SNOWFLAKE_WAREHOUSE` to Vercel env vars.
