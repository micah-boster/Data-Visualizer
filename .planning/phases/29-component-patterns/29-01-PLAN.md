**Project constraints (from CONTEXT + RESEARCH + STATE):**
- Single canonical size only — no compact/hero variants.
- Value on its own line; trend on a SECOND line below with explanatory phrase. Cards grow slightly taller.
- Label uses `.text-label uppercase text-muted-foreground` (overline recipe — Phase 27-06 canonical).
- Optional `icon?: ReactNode` slot LEFT of the label text (Lucide h-4 w-4 text-muted-foreground).
- Polarity-aware trend colors: reuse getPolarity(), don't re-implement.
- Stale and comparison are PROP SURFACE ONLY per resolved decision #1 and #2. No data plumbing — StatCard accepts the props and renders correctly when passed. Document the deferred plumbing inline in the file via JSDoc above the prop.
- No motion beyond Skeleton's animate-pulse — Phase 30 formalizes motion.
- Pattern file MUST pass check:tokens (no raw text-* classes, no paired font-medium + token).
- Pattern file MUST pass check:surfaces (use shadow-elevation-raised, not raw shadow-sm/md).
- Trend arrow is a glyph, not a digit — use `.text-label` (weight 500 baked in) rather than `font-medium + text-caption`. See STATE Phase 27-03 and 27-06 for the carve-out rationale.
- Em-dash uses Unicode escape `'\u2014'` (NOT raw `"—"` — matches KpiCard today, avoids codepoint linter flags).
