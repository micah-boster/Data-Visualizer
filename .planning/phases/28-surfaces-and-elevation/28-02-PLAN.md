Rationale per swap:
- sticky top-0 z-20 — header pins to viewport top. z-20 sits above content (implicit z-0) and below popovers (z-50 per Base UI convention). No existing z conflicts per 28-RESEARCH Pitfall 2 audit.
- bg-surface-raised -> bg-surface-translucent — color-mix semi-opaque backdrop; with backdrop-blur-md this creates the glass effect. `--surface-translucent` is derived from `--surface-raised` at 80/82% (light/dark), so at rest the header color reads close to raised; the blur difference appears when content scrolls underneath.
- Add backdrop-blur-md — Tailwind v4 built-in GPU-accelerated blur. md (12px) is the right default for 40px-tall chrome; lg/xl would make dense table data unreadable.
- shadow-xs -> shadow-elevation-chrome — semantic 3-layer (key + ambient + rim). Rim highlight carries top-edge separation per CONTEXT lock.

DO NOT re-add border-b. 26-03 dropped it intentionally; this plan continues the shadow-only discipline.

DO NOT touch children of <header> (freshness indicator, theme toggle, etc.). Root className only.

If pilot visual testing reveals the blur is too strong or dark-mode shadow reads heavy, note the specific observation in SUMMARY Deviations — do NOT hand-roll opacity; instead flag as a follow-up tune against 28-01 token values.
