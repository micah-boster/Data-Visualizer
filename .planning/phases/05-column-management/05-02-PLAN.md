**Create `src/components/table/draggable-header.tsx`:**
- Component wraps a single `<th>` table header cell with drag support.
- Uses `useSortable({ id: header.id })` from @dnd-kit/sortable.
- Renders existing header content (sort click, resize handle) PLUS a drag handle.
- Drag handle: lucide `GripVertical` icon, only visible on hover (opacity-0 group-hover:opacity-100). The grip triggers drag via `{...listeners}` on just the grip div, NOT the entire header (so click-to-sort still works on the rest of the header).
- Identity/pinned columns: if `column.getIsPinned()` returns truthy, render without sortable (no grip, no drag). Use a conditional: pinned columns render a plain `<th>`, non-pinned get the sortable wrapper.
- Apply `transform` and `transition` from useSortable via `CSS.Translate.toString(transform)`.
- When `isDragging` is true, set `opacity: 0.5` on the cell (the ghost effect).
- Preserve all existing header behavior: pinning styles from `getCommonPinningStyles`, numeric right-alignment, sort indicator, resize handle.

**Update `src/components/table/table-header.tsx`:**
- Wrap the header row's non-pinned cells in a `SortableContext` with `horizontalListSortingStrategy`.
- The items array for SortableContext: `headerGroup.headers.filter(h => !h.column.getIsPinned()).map(h => h.id)`.
- Replace the current `<th>` rendering with `DraggableHeader` component for each header.
- Pinned headers render before the SortableContext as plain `<th>` elements (or DraggableHeader with drag disabled).

**Update `src/components/table/data-table.tsx`:**
- Wrap the `<table>` element in `DndContext` from @dnd-kit/core with `closestCenter` collision detection.
- Handle `onDragEnd` event: extract `active.id` and `over.id`, find their indices in `columnOrder`, compute new order via `arrayMove`, call `setColumnOrder`.
- Add a `DragOverlay` component that renders a styled header cell preview during drag. The overlay should show the column label text with a blue-tinted background and slight shadow to indicate it's being dragged. Set explicit width matching the column's current size.
- IMPORTANT: DndContext must wrap only the table area, not the entire component (to avoid interfering with sidebar drag, which is handled separately).
  </action>

  </verify>
  <done>
    Column headers are draggable via grip handle. Pinned/identity columns are excluded from drag. Drag shows ghost effect and overlay. Column order updates on drop and persists via useColumnManagement/localStorage.
