**Create `src/lib/columns/filter-functions.ts`:**
Two custom TanStack Table `FilterFn` implementations:

1. `checklistFilter`: receives `filterValue: string[]`. If empty array, return true (no filter). Otherwise check if `String(row.getValue(columnId))` is included in the array.

2. `rangeFilter`: receives `filterValue: { min?: number; max?: number }`. If both undefined, return true. Check `row.getValue<number>(columnId)` against bounds. Treat null cell values as non-matching (return false).

Both functions should have proper TypeScript typing using `FilterFn<Record<string, unknown>>` from @tanstack/react-table.

**Create `src/hooks/use-column-filters.ts`:**
- Manages in-column filter state as React state (NOT URL-backed -- these are session-only).
- State shape: `Record<string, unknown>` mapping column IDs to their filter values (string[] for text, {min?, max?} for numeric).
- `setColumnFilter(columnId: string, value: unknown)` -- sets or clears a single column's filter.
- `clearColumnFilter(columnId: string)` -- removes the filter for a column.
- `clearAllColumnFilters()` -- resets all.
- `activeColumnFilters`: array of `{ columnId: string; label: string; displayValue: string }` for chip display. For text checklist: "N values selected". For numeric range: "min-max" or ">min" or "<max".
- `mergedColumnFilters(dimensionFilters: ColumnFiltersState): ColumnFiltersState` -- takes the URL-backed dimension filters from useFilterState and merges in-column filters on top. This is the single ColumnFiltersState passed to TanStack Table. CRITICAL: dimension filter entries use exact string match (existing behavior). In-column filters use the custom filterFn. The merge concatenates both arrays. TanStack ANDs all entries automatically.

**Update `src/lib/columns/definitions.ts`:**
- Import `checklistFilter` and `rangeFilter` from filter-functions.
- For each column definition, assign `filterFn`:
  - Text columns (type 'text'): `filterFn: checklistFilter`
  - Numeric columns (type 'currency', 'percentage', 'count', 'number'): `filterFn: rangeFilter`
  - Date columns: skip filterFn (no in-column filter for dates in this phase)
- Also set `enableColumnFilter: true` on filterable columns and `enableColumnFilter: false` on identity columns (identity columns already have dimension filters from Phase 4).

**Update `src/lib/table/hooks.ts`:**
- Accept `columnFilters` as the merged result from useColumnFilters (already happens via the existing parameter, no change needed if the merged array is passed in from data-table).
- Ensure `getFilteredRowModel` is still enabled (it already is).

**Update `src/components/table/data-table.tsx`:**
- Import and use `useColumnFilters` hook.
- Replace the raw `columnFilters` from `useFilterState` with the merged result: call `mergedColumnFilters(dimensionColumnFilters)` and pass that to `useDataTable`.
- This is the critical integration point where Phase 4 dimension filters and Phase 5 in-column filters compose.
