- Digital Channels (6 cols) -- all OUTBOUND_*/INBOUND_* rate columns

Export `COLUMN_GROUPS` array, `ColumnGroup` type, and `getDefaultVisibility()` function that returns a VisibilityState with ~12-15 key columns visible (identity + TOTAL_ACCOUNTS, TOTAL_AMOUNT_PLACED, AVG_AMOUNT_PLACED, TOTAL_COLLECTED_LIFE_TIME, TOTAL_ACCOUNTS_WITH_PAYMENT, COLLECTION_AFTER_3_MONTH, COLLECTION_AFTER_6_MONTH, COLLECTION_AFTER_12_MONTH, PENETRATION_RATE_POSSIBLE_AND_CONFIRMED). Also export `getDefaultColumnOrder()` returning all column keys in their natural config.ts order.

**Create `src/lib/columns/persistence.ts`:**
- `STORAGE_KEY = 'bounce-dv-columns'`
- `ColumnState` type: `{ visibility: VisibilityState; order: string[] }`
- `loadColumnState(): ColumnState | null` -- reads from localStorage, returns null if missing/corrupt (try-catch)
- `saveColumnState(state: ColumnState): void` -- writes to localStorage (try-catch, silent fail)

**Create `src/hooks/use-column-management.ts`:**
- Hook that manages `columnVisibility` (VisibilityState) and `columnOrder` (string[])
- On mount: try `loadColumnState()`. If found, use it. Otherwise use `getDefaultVisibility()` and `getDefaultColumnOrder()`.
- IMPORTANT: Initialize with defaults in useState, then apply localStorage in useEffect to avoid Next.js hydration mismatch.
- On change: `saveColumnState()` in a separate useEffect watching visibility + order
- Expose: `columnVisibility`, `setColumnVisibility`, `columnOrder`, `setColumnOrder`, `resetToDefaults()`, `visibleCount` (number of true entries)
- `toggleColumn(key: string)` -- toggles visibility, skips if identity
- `toggleGroup(groupKey: string, visible: boolean)` -- sets all non-identity columns in group
- `showAll()` / `hideAll()` -- all non-identity columns shown/hidden
- Interact with preset system: when user manually toggles columns, set activePreset to 'custom' (not one of the named presets). Accept `setActivePreset` callback as parameter.

**Update `src/lib/table/hooks.ts`:**
- Extract `columnVisibility` / `setColumnVisibility` state management OUT of useDataTable. Instead, accept them as parameters in `UseDataTableOptions`: `columnVisibility?: VisibilityState`, `onColumnVisibilityChange?: OnChangeFn<VisibilityState>`, `columnOrder?: string[]`, `onColumnOrderChange?: OnChangeFn<string[]>`.
- If not provided, fall back to the existing preset-based default behavior (backward compatible).
- Add `onColumnOrderChange` setter to the useReactTable config.
- Return `columnOrder` and `setColumnOrder` from the hook alongside existing returns.
