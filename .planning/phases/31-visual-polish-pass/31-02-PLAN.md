<objective>
Build the canonical focus-glow language for the app and migrate the legacy hand-rolled focus rings onto it. DS-31 closes the "single focus language" CONTEXT lock.

Purpose: The app currently has ~23 focus-visible sites across 10 files with inconsistent recipes (some `focus-visible:ring-3 ring-ring/50`, some `focus:ring-2` legacy, some `focus-visible:ring-[3px]`). Net-new: `focus-within` rings on ToolbarGroup cluster + saved-view rows (no prior treatment — Pitfall 2 requires `:has(:focus-visible)` over `:focus-within` to avoid mouse-click flicker).

```css
/*
 * Phase 31 DS-31: Single focus-glow language.
 * - .focus-glow on individual focusable elements (inputs, buttons, selects)
 * - .focus-glow-within on interactive containers (ToolbarGroup cluster, saved-view row)
 * - Uses :has(:focus-visible) on the within variant so mouse clicks do NOT fire the glow (31-RESEARCH Pitfall 2)

  outline: 2px solid var(--ring);
  outline-offset: 2px;
  box-shadow: 0 0 0 3px color-mix(in oklch, var(--ring) 30%, transparent);
}
```
