Seven negative checks:

1. **Raw border hex/rgb/oklch colors** outside allowlist:
   Pattern: `border-\\[(#|rgb|oklch|var\\()`
   Message: "Raw border color literals outside allowlist. Use border-border (consumes --border after 31-01 retune)."

2. **Border-opacity overrides on design tokens**:
   Pattern: `border-(border|input|sidebar-border)/[0-9]+`
   Message: "border-opacity override on a design token. Use the token as-is; retune the token in globals.css if needed."

3. **Raw Tailwind palette border colors**:
   Pattern: `border-(red|emerald|amber|blue|green|slate|zinc|gray|yellow|orange|rose|purple|sky|cyan|teal|pink|violet|indigo|lime|fuchsia)-[0-9]+`
   Message: "Raw Tailwind palette border color. Use semantic state tokens (border-destructive, border-success, etc.) or border-border."

4. **Raw focus-visible ring recipes** (outside shadcn):
   Pattern: `focus-visible:ring-\\[|focus:ring-[0-9]`
   Message: "Raw focus ring recipe outside shadcn primitives. Use .focus-glow or .focus-glow-within."

5. **Unscoped ::-webkit-scrollbar** outside globals.css + ui/ allowlist:
   Pattern in non-ui/, non-globals.css files: `::-webkit-scrollbar`
   Message: "Raw ::-webkit-scrollbar outside globals.css. Use .thin-scrollbar utility."

6. **Raw linear-gradient(transparent, ...) divider patterns** outside allowlist:
   Pattern: `linear-gradient\\(to (right|bottom),\\s*transparent,`
   Message: "Raw gradient divider outside allowlist. Use .divider-horizontal-fade / .divider-vertical-fade utilities or <SectionDivider />."

7. **Raw inset top-edge shadow hardcodes**:
   Pattern: `inset[_ ]0[_ ]1px[_ ]0[_ ]0[_ ](rgb|oklch)`
   Message: "Raw inset top-edge shadow. Glass highlight lives in --shadow-elevation-raised (dark) — do not stack a second shadow utility (Pitfall 1)."

8. **Bonus negative — transition-shadow on focus-glow** (Pitfall from 31-RESEARCH §DS-31):
   Pattern in files that reference .focus-glow: `transition[-:][^;]*shadow`
   Scope narrow — only where focus-glow is the containing context. Simpler implementation: add a global check for `.focus-glow` or `.focus-glow-within` appearing on the same line as `transition-shadow` or `transition-[box-shadow]`.
   (Optional — land if easy; else document as manual-review in SUMMARY.)

**package.json**: Under the `"scripts"` block, add: `"check:polish": "bash scripts/check-polish.sh",` alongside the existing check:tokens / check:surfaces / check:components / check:motion entries.

Make the script executable: `chmod +x scripts/check-polish.sh`.

Run `npm run check:polish` on the current tree — it MUST exit 0 (all Phase 31-01..31-05 work used the utilities, not raw patterns).
