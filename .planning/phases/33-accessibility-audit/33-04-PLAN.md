<task type="auto">
  <name>Task 2: Atomic token retune loop — adjust, verify, commit, repeat</name>
  <files>
    src/app/globals.css
  </files>
  <action>
For each (fg-token, bg-token, mode) triple from Task 1's plan:

1. **Propose new value** that meets AA:
   - Use https://www.myndex.com/APCA/ or an oklch-aware contrast calculator to pick a value.
   - Prefer lightness / chroma adjustments over hue changes. For brand-green pairs, preserve hue; only shift L/C.
   - In dark mode, raising `--muted-foreground` L is usually enough (e.g., `oklch(0.76 0.012 60)` → `oklch(0.80 0.012 60)` or `0.82` — iterate).
   - In light mode, lowering the muted-foreground L darkens text — easier.

2. **Edit `globals.css`**:
   - Update the variable in `:root` and/or `.dark` block (whichever mode needed the fix)
   - Inline comment: `/* A11Y-04: bumped from <old> to <new> for 4.5:1 AA on muted-foreground on surface-base */`
   - Keep variable name and block structure unchanged — just the value changes

3. **Re-run `npm run check:a11y`**:
   - Confirm the targeted pair now passes (that specific violation.id disappears from the results)
   - Confirm no OTHER contrast pair went RED as collateral (RESEARCH Pitfall 2 cascade) — if it did, the new value is wrong; iterate
   - Confirm other guards (check:tokens / surfaces / components / motion / polish) all still green — token retune shouldn't regress visual

4. **Atomic commit**:
   - One variable edit = one commit per token (per mode)
   - Commit message: `fix(33-04): retune --<token> in <mode> mode for WCAG AA contrast`

5. Iterate over all pairs from Task 1. If the count is ≤3 distinct token edits, this task completes in one sitting. If more, work through them sequentially.

6. **Special case — brand-green / primary-foreground**:
   - CONTEXT locks "preserve brand intent." If primary button contrast fails, DO NOT mute the green. Instead:
     - Light mode: ensure `--primary-foreground` is light enough (neutral-50 is already 0.985)
     - Dark mode: verify `--primary` vs `--primary-foreground` pairing. If the dark-mode button text is too dark on the brand-green bg, darken `--primary-foreground` OR lighten `--primary`.
   - If neither adjustment meets AA without compromising brand, document the tradeoff in SUMMARY and escalate as a USER DECISION at Plan 05 close-out. Do NOT silently choose a value.

7. **Placeholder text**:
   - `placeholder:text-muted-foreground` in inputs is a common AA fail. Fix at token level (same --muted-foreground retune) OR if a placeholder-specific token exists, adjust that.
   - Verify WebKit's ~45% opacity dimming (default) isn't pushing it further below AA — browsers apply their own placeholder style on top of our color.

8. Continue until `npm run check:a11y` shows ZERO color-contrast violations across all 4 routes × both themes.
