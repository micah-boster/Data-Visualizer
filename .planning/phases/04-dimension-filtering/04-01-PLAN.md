  <name>Task 1: Create useFilterState hook for URL-backed filter state</name>
  <files>src/hooks/use-filter-state.ts</files>
  <action>
Create a `useFilterState` hook that reads/writes filter state from URL search params.

**Implementation:**
- Import `useSearchParams`, `usePathname`, `useRouter` from `next/navigation`.
- Define param-to-column mapping: `partner` -> `PARTNER_NAME`, `type` -> `ACCOUNT_TYPE`, `batch` -> `BATCH`.
- `columnFilters` (ColumnFiltersState): Derived via `useMemo` from `searchParams.toString()` (use the string as dependency, not the object, to avoid reference-change re-renders). For each param in the mapping, if `searchParams.get(param)` is non-null, push `{ id: columnId, value }` into the array.
- `setFilter(param: string, value: string | null)`: Creates new `URLSearchParams` from current, sets or deletes the param, calls `router.replace(pathname + '?' + params.toString())`. Uses `replace` not `push` to avoid history pollution.
- `clearAll()`: Calls `router.replace(pathname)` to strip all filter params.
- Expose `activeFilters` as a convenience: array of `{ param, columnId, label, value }` objects for chip rendering. Labels: partner -> "Partner", type -> "Account Type", batch -> "Batch".
- Export the hook and the `FILTER_PARAMS` mapping (other components need to know param keys).

**Auto-clear cascading batch:** When `setFilter('partner', value)` is called, also check if current batch param still makes sense. Accept an optional `data` parameter (the full dataset array) so the caller can pass it for cascade validation. If partner changes and current batch is set, check if any row with the new partner value has the current batch value. If not, auto-delete the batch param in the same URL update. This prevents zero-result selections (locked decision).

**Key constraint:** Single source of truth is URL params. No `useState` for filter values.
