# Phase 27 Plan 07: /tokens Numeric Demo Alignment Fix

**Fixes UAT Gap #1 from Phase 27 verification: the three-row numeric demo on /tokens renders left-aligned with misaligned decimals, undermining the point of the tabular-nums demonstration.**

## Context

UAT test 2 surfaced that the three-row numeric demo block at `src/components/tokens/type-specimen.tsx:209-213` renders left-aligned. The author padded with leading whitespace in text content (`"  987,654.32"`), but HTML collapses leading whitespace in text nodes, so all three rows render flush-left at the container edge. Even though `tabular-nums` is baked into `.text-body-numeric`, decimal alignment requires the right edges of the rows to line up.

Fix is trivial: add `text-right` to the parent container and drop the cosmetic leading spaces.

## Task 1 — Right-align the numeric variants inset demo

**File:** `src/components/tokens/type-specimen.tsx`

**Current (lines 209-213):**

```tsx
<div className="flex flex-col gap-stack p-card-padding bg-surface-inset rounded-lg">
  <div className="text-body-numeric">1,234,567.89</div>
  <div className="text-body-numeric">  987,654.32</div>
  <div className="text-body-numeric">    4,321.10</div>
</div>
```

**Target:**

```tsx
<div className="flex flex-col gap-stack p-card-padding bg-surface-inset rounded-lg text-right">
  <div className="text-body-numeric">1,234,567.89</div>
  <div className="text-body-numeric">987,654.32</div>
  <div className="text-body-numeric">4,321.10</div>
</div>
```

**Changes:**
1. Append `text-right` to the parent div's className.
2. Remove leading whitespace from the text content of the second and third rows (was `"  987,654.32"` / `"    4,321.10"`, becomes `"987,654.32"` / `"4,321.10"`).

## Verification

   - Numbers are right-aligned at the container's right edge.
   - Decimal points visibly line up vertically across `1,234,567.89` / `987,654.32` / `4,321.10`.
3. Light + dark mode visual sanity (matches feedback_testing memory).
