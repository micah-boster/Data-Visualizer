1. Create `src/lib/ai/context-builder.ts`:

This module builds a focused data summary string for Claude's system prompt based on the current drill level. It receives pre-computed data (partner stats, anomalies) and formats it — it does NOT query Snowflake or read files.

**Three context levels (from CONTEXT.md locked decisions):**

- **Root level**: For each partner, include: name, batch count, key aggregates (penetration rate, 6mo/12mo collection rates, total collected, total placed), anomaly status (flagged/not, severity score, flagged batch count). Format as a compact table or structured list.

- **Partner level**: Full detail for the current partner (all KPI aggregates, all batch anomaly details). Plus a one-line portfolio rank summary for each metric (e.g., "3rd of 8 partners on penetration rate"). Include a note: "Future enhancement: all-partners-always context."

- **Batch level**: Batch aggregates (placed, collected, penetration, collection rates). Parent partner summary (name, overall KPIs). Top 5-10 outlier accounts if account data is available (highest balance, worst performing).

**Format choice (Claude's discretion):** Use hybrid JSON/markdown. Structured data as compact JSON for precise numbers; section headers and labels as markdown for readability.

**Token budget awareness:** Each partner summary should be ~100-150 tokens. Root level with 8-10 partners = ~1000-1500 tokens. Partner detail = ~500-800 tokens. Batch detail = ~300-500 tokens. Total data context target: under 2000 tokens.

The function signature:
```typescript
export function buildDataContext(
  drillState: { level: DrillLevel; partnerId: string | null; batchId: string | null },
  data: {
    partners: Array<{ name: string; stats: KpiAggregates; batchCount: number }>;
    anomalies: Map<string, PartnerAnomaly>;
    currentPartnerStats?: PartnerStats;
    accountOutliers?: Array<{ accountId: string; balance: number; penetration: number }>;
  }
): string
```

Returns a formatted string ready to embed in the system prompt. If data is empty or unavailable, return a string saying "No data currently loaded."

**System prompt structure:**

```
You are a data analyst assistant for the Bounce partnerships team. You help team members understand their debt collection portfolio performance data.

## Your Personality
- Smart colleague: conversational but informed. Not a stiff analyst briefing.
- When you answer a question, add one relevant follow-up insight if something notable exists in the data.
- Adaptive length: simple lookups get 1-2 sentences, complex analytical questions get a short paragraph.
- Always cite exact numbers: "Penetration is 12.3%, down 1.8pp" — never vague directional language.

## Current View
The user is viewing: [drill level description based on drillState]
[If filters active: "Active filters: [filter description]"]

## Available Data
[dataContext string inserted here]

## Critical Rules
1. ONLY reference data provided above. If the user asks about data not shown here, respond: "I don't have data on [topic]. I can only answer questions about [what's available]."
2. Never speculate, approximate, or infer data that isn't explicitly provided.
3. Light operational recommendations are OK ("this declining trend might warrant a check-in") but never give financial advice.
4. If the user asks a non-data question, answer briefly if harmless, then redirect: "By the way, I can also help with questions about your portfolio data."
```

**Drill state description:**
- Root: "the portfolio overview (all partners)"
- Partner: "Partner: {partnerId}"
- Batch: "Batch: {batchId} (Partner: {partnerId})"
