Create `src/hooks/use-chart-presets.ts` — mirror `src/hooks/use-saved-views.ts` 1:1 for hydration structure. Start with the `'use client'` directive.

Structure:
1. `useState<ChartPreset[]>([])` — empty-init for SSR safety. NEVER read localStorage during render (KI-13 precedent).
2. `useRef<boolean>(false)` named `hasHydrated`.
3. First `useEffect`: load user presets via `loadChartPresets()`, sanitize (drop any whose `.definition` fails `chartDefinitionSchema.safeParse` — mirrors Phase 34 `sanitizeSnapshot` drop-unknown-listIds pattern), then `setPresets([...BUILTIN_PRESETS, ...sanitizedUser])`, then `hasHydrated.current = true`. Deps: `[]` (built-ins don't change; user presets change via local state only).
4. Second `useEffect`: when `hasHydrated.current === true`, call `persistChartPresets(presets.filter(p => !p.locked))` — built-ins NEVER persist. Deps: `[presets]`.
5. CRUD methods wrapped in `useCallback`:
   - `savePreset(name, definition)`: trims name; creates `{ id: crypto.randomUUID(), name, locked: false, createdAt: Date.now(), definition: structuredClone(definition) }` (deep-copy per Pitfall 4 from research — prevents reference leak to caller state); appends via `setPresets`; returns the new ChartPreset synchronously.
   - `deletePreset(id)`: look up in current state; return `undefined` if not found OR if `locked === true` (refuses to delete built-ins); else splice out and return the deleted entry.
   - `restorePreset(preset)`: append as-is via `setPresets` (used by undo-toast).
   - `hasPresetWithName(name)`: case-insensitive, trimmed comparison across BOTH built-ins and user presets.
   - `renamePreset(id, name)`: no-op if `locked === true`; otherwise update in place with a fresh `updatedAt`-style — actually just overwrite `name` (ChartPreset does not have updatedAt; keep the entity minimal).
   - `getPreset(id)`: find by id across merged list.
6. Return the object exactly shaped per the interfaces block.

Hydration-vs-seed note in a JSDoc block above the hook body: reference Phase 35-02 Pitfall 2 — downstream human-verify for stale-column banner CANNOT rely on seeding localStorage and reloading, because this same hydration-then-persist shape will overwrite the seed.
