  <action>
Create `src/lib/charts/stale-column.ts`:
- Import `COLUMN_CONFIGS` and `ColumnConfig` from `@/lib/columns/config`, `getEligibleColumns`, `isColumnEligible` from `@/lib/columns/axis-eligibility`.
- Export `resolveColumnWithFallback(chartType, axis, requested)`:
  - `requested === null` → return `null` (signals "axis not picked yet"; caller renders EmptyState).
  - Look up `COLUMN_CONFIGS.find(c => c.key === requested.column)`:
    - Found AND eligible under `(chartType, axis)` via `isColumnEligible` → `{ config: found, stale: false, requested: requested.column }`.
    - Found but NOT eligible (e.g. a stored numeric column after a switch to bar-X) → treat as stale: pick the first eligible column via `getEligibleColumns(chartType, axis)[0]`, return `{ config: fallback, stale: true, requested: requested.column }`. If no eligible column exists, return `null`.
    - Not found in registry → same stale path with fallback.
- Pitfall 3 lock: this function is pure/read-only. Never dispatches onDefinitionChange. The renderer shows the banner and uses the fallback config; the user's explicit column pick (via the toolbar) is what eventually overwrites the stale ref.

Create `src/lib/charts/stale-column.smoke.ts`:
- Node `--experimental-strip-types` + `node:assert/strict` (explicit `.ts` imports).
- Assert resolveColumnWithFallback:
  - `requested: null` → returns `null`.
  - Line X with `{ column: 'BATCH_AGE_IN_MONTHS' }` → `{ stale: false, config.key === 'BATCH_AGE_IN_MONTHS' }`.
  - Line X with `{ column: 'NONEXISTENT_KEY' }` → `{ stale: true, requested: 'NONEXISTENT_KEY', config.key === /* first eligible key */ }`.
  - Bar X with `{ column: 'TOTAL_COLLECTED_LIFE_TIME' }` (numeric — not eligible for bar-X which wants categorical) → `{ stale: true }` with a text-type fallback config.
  - Scatter Y with `{ column: 'TOTAL_COLLECTED_LIFE_TIME' }` → `{ stale: false }` (numeric is eligible for Y under any chart type).

   - Variant-specific primitive:

     - scatter: `<Scatter dataKey={yCol.key} />` with `data={rows}`.
     - bar: `<Bar dataKey={yCol.key} />`.
