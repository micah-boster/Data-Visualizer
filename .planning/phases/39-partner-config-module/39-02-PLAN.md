<objective>
Add the per-pair segment configuration substrate: schema, localStorage storage, React hook + context, evaluator, and the Partner Setup slide-over Sheet editor reachable from the sidebar context menu.

Purpose: PCFG-05 requires an editable segment list persisted in localStorage for each (partner, product) pair so the partnerships team can model sub-cohorts (e.g. Snap English/Spanish, Happy Money bank A/B) once those columns land in the data. This plan ships the full UI and storage layer so Plan 39-04 can wire charts/KPIs onto the evaluator. Mirrors the `partner-lists/*` pattern exactly — no new abstractions.

Output: `src/lib/partner-config/*` new domain, `usePartnerConfig` hook + context, `PartnerSetupSheet` slide-over editor accessible via right-click on any sidebar pair row. Staged-edit semantics (explicit Save / Cancel) matching `CreateListDialog`. Empty-state messaging when no viable segmenting columns exist in the dataset (today's reality on `agg_batch_performance_summary`).
</objective>
