<objective>
Lock the canonical domain vocabulary in code and surface it in-product. Ship three artifacts the rest of the milestone (and v5.0) builds on:

1. **docs/GLOSSARY.md** — paragraph-length glossary grouped by domain area, written for a new analyst joining the team. Doubles as onboarding material.
2. **src/lib/vocabulary.ts** — `TERMS` registry exporting `{ label, definition, synonyms, seeAlso }` per term. The single rename point for the app's vocabulary.
3. **src/components/ui/term.tsx** — `<Term name="...">` component that surfaces the registry's one-sentence definition on hover/focus with click-to-pin.

Then wire `<Term>` into the four primary surfaces specified by CONTEXT.md and the roadmap (KPI card labels, **table headers**, breadcrumb segments, sidebar section labels) using the **first-instance-per-surface** rule.

Purpose: v5.0 introduces 5 new domain terms. Without a registry and a rendering primitive in place first, those terms layer in as more ad-hoc strings and the conceptual sprawl compounds. This plan ships the substrate so v5.0 phases extend `TERMS` as part of their plan deliverable.

Output: Glossary doc, registry module, `<Term>` component, smoke test, and four wired consumer surfaces (KPI cards, breadcrumb, sidebar, table headers). Zero behavior change for users who never hover.
</objective>

**Behavior contract (from CONTEXT.md):**

```typescript
export interface TermProps {
  /** The term key in TERMS registry. Type-safe via TermName. */
  name: TermName;
  /**
   * The rendered text — usually the term itself, but the registry's `label` is also valid.
   * Children let callers control casing/pluralization (e.g. "Batches" but name="batch").
   */
  children: React.ReactNode;
  /** Optional className override (forwarded to the trigger span). */
  className?: string;
}

export function Term({ name, children, className }: TermProps): JSX.Element;
```

**Implementation:**

1. **At rest:** Render a plain `<span>` with `children`. NO visual treatment until hover/focus. CONTEXT.md locks this — the underline appears only on hover/focus.
2. **On hover/focus:** Apply `border-b border-dotted border-muted-foreground` (subtle dotted underline) and `cursor-help`. Use Tailwind's `hover:` and `focus-visible:` pseudo-class chain so keyboard users get the same treatment.
3. **Hover popover:** After ~400ms hover delay, surface a popover with the term's `definition` (h3-equivalent label as title, paragraph for definition, optional "See also: …" footer linking to other Terms). Use the existing `Popover` primitive from `src/components/ui/popover.tsx`. Popover content uses `bg-surface-overlay` + `shadow-elevation-overlay` (Phase 28 tokens) — the primitive already does this.
4. **Click-to-pin:** Default popover is hover-controlled (Base UI Popover supports `openOnHover` / `delay` props — verify in node_modules/@base-ui/react/popover docs; if not, manage open state manually with onMouseEnter/onMouseLeave + clickToPin flag). Click toggles a "pinned" mode where the popover stays open until Esc or click-outside.
5. **Mobile / touch:** Tap-to-pin (no hover state). Detect via `(pointer: coarse)` media query OR simply rely on touch fires `onClick` and skip hover handlers — Base UI tooltips/popovers handle this automatically; keep it simple.
6. **Keyboard:** Trigger span has `tabIndex={0}` so keyboard users can focus it. Enter/Space pin/unpin (matches click semantics). Esc dismisses pinned popover.
7. **ARIA:** Use the existing Popover primitive's built-in ARIA wiring (the `data-slot="popover-trigger"` / `data-slot="popover-content"` pattern). Add `aria-describedby` only if the primitive doesn't supply equivalent.
8. **Cross-references:** If `seeAlso.length > 0`, render a "See also: " line at the popover bottom, with each related term name as a clickable button that updates the popover content to that term (re-uses the same component recursively, OR simpler: just render them as plain comma-separated text. Pick the simpler one for v1 — comma-separated text. Add interactivity in a fast-follow if user feedback wants it).

**Type tokens (Phase 27 — `src/components/ui/**` IS in the allowlist BUT prefer named tokens where possible):**
- Popover title: `text-title` (NOT text-heading — popover titles are not section anchors per Phase 27-04 precedent)

**Avoid:** Adding analytics / hover tracking — explicitly deferred per CONTEXT.md "Deferred Ideas".

Open `node_modules/@base-ui/react/dist/popover/index.d.ts` (or the package types) to confirm hover-open / openOnHover / delay prop names before committing the implementation. If Base UI Popover does NOT support hover-open, manage the `open` state with a controlled `useState` plus `onMouseEnter` (set true after 400ms timeout), `onMouseLeave` (clear timeout, set false unless pinned), `onClick` (toggle pinned).
