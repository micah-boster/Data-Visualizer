<task type="auto">
  <name>Task 1: Create CSV generation utility and browser download trigger</name>
  <files>src/lib/export/csv.ts, src/lib/export/download.ts</files>
  <action>
Create two pure utility files in `src/lib/export/`.

**csv.ts — `buildCSVFromTable(table, activeFilters)`:**
- Accept a TanStack Table instance (`Table<Record<string, unknown>>`) and an `ActiveFilter[]` array
- Read visible columns via `table.getVisibleLeafColumns()` — skip any column where `col.id` starts with an internal prefix (unlikely but defensive)
- Build header row using `col.columnDef.header` if it is a string, otherwise fall back to `col.id`
- Build metadata rows at the top of the CSV:
  - Row 1: `"Source","agg_batch_performance_summary"`
  - Row 2: `"Exported","YYYY-MM-DD HH:MM:SS"` (local time, formatted with Intl.DateTimeFormat)
  - Row 3: `"Active Filters","Partner: Acme; Account Type: Credit Card"` or `"Active Filters","None"` — derive from activeFilters array using `${f.label}: ${f.value}` joined with `; `
  - Row 4: `"Column Mapping","Partner = PARTNER_NAME | Lender ID = LENDER_ID | ..."` — map each visible column's display name to its Snowflake key using the COLUMN_CONFIGS lookup. This is for traceability.
  - Row 5: blank line separator
- Build data rows from `table.getRowModel().rows` (this already reflects filtering + sorting):
  - For each row and each visible column, get the raw value via `row.getValue(col.id)`
  - If value is null/undefined, output empty string
  - If column meta.type is a numeric type (use `isNumericType` from formatting module), apply `getFormatter(meta.type)` to produce the formatted string
  - If column meta.type is 'date', apply `formatDate`
  - If column meta.type is 'text', use `String(value)`
- All values (including headers) must go through `escapeCSV()` before joining with commas
- `escapeCSV(value)`: if value contains comma, double-quote, or newline, wrap in double quotes and escape internal quotes by doubling them (RFC 4180)
- Return `{ csv: string, rowCount: number }` — rowCount is `table.getRowModel().rows.length` (for the toast)
- Export both `buildCSVFromTable` and `escapeCSV`

**download.ts — `downloadCSV(csvContent, filename)` and `getExportFilename()`:**
- `downloadCSV`: Prepend UTF-8 BOM (`\uFEFF`) for Excel compatibility. Create Blob with type `text/csv;charset=utf-8;`. Create temporary anchor element, set href to `URL.createObjectURL(blob)`, set download attribute to filename, append to body, click, remove from body, revoke object URL.
- `getExportFilename`: Return `bounce-batch-performance-YYYY-MM-DD.csv` using `new Date().toISOString().split('T')[0]`
