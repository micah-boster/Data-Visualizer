  <action>
    1. Run `pnpm create next-app@latest . --yes` from the project root to scaffold into the current directory. Use the default options (App Router, TypeScript, Tailwind CSS, src/ directory, ESLint).

    2. Initialize shadcn/ui: `pnpm dlx shadcn@latest init` -- accept defaults. This sets up Tailwind CSS vars, cn() utility, and components.json.

    3. Install core dependencies:
       `pnpm add snowflake-sdk @tanstack/react-query next-themes zod`

    4. Install shadcn/ui components needed for Phase 1:
       `pnpm dlx shadcn@latest add button card skeleton alert sidebar separator`

    5. Create `.env.example` (committed to repo) with all Snowflake credential placeholders:
       ```
       SNOWFLAKE_ACCOUNT=
       SNOWFLAKE_USERNAME=
       SNOWFLAKE_PASSWORD=
       SNOWFLAKE_WAREHOUSE=
       SNOWFLAKE_DATABASE=
       SNOWFLAKE_SCHEMA=
       SNOWFLAKE_ROLE=
       ```

    6. Ensure `.gitignore` includes `.env.local` and `.env*.local` (create-next-app should handle this, but verify).

    7. Verify the dev server starts with `pnpm dev` and the default Next.js page renders.

    IMPORTANT: Do NOT create `.env.local` with actual credentials -- that is a user action. Only create `.env.example` as a template.
  </action>
  <verify>
    `pnpm dev` starts without errors; visiting localhost:3000 shows the Next.js default page; `cat .env.example` shows all 7 SNOWFLAKE_ vars; `.env.local` is in `.gitignore`; `pnpm list snowflake-sdk @tanstack/react-query next-themes zod` shows all installed.
