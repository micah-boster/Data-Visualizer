    2. Create `src/lib/metabase-import/parse-metabase-sql.ts` — one exported function that wraps `node-sql-parser/build/snowflake`. Structure:

       a. **Preprocess Metabase template tags FIRST** (Pitfall 1):
          ```typescript
          function stripMetabaseTemplates(sql: string): string {
            // [[ ... ]] optional clauses removed entirely (we have no params to resolve them).
            // {{var}} replaced with NULL so WHERE remains parseable; resulting filter will
            // land in skippedFilters with a type-mismatch reason the preview surfaces.
            return sql.replace(/\[\[[\s\S]*?\]\]/g, '').replace(/\{\{[^}]+\}\}/g, 'NULL');
          }
          ```

       b. **Parse with try/catch**, capture `.location` if available:
          ```typescript
          import { Parser } from 'node-sql-parser/build/snowflake';
          const parser = new Parser();
          let ast: unknown;
          try {
            ast = parser.astify(stripMetabaseTemplates(sql), { database: 'Snowflake' });
          } catch (err) {
            const e = err as { message?: string; location?: { start?: { line?: number; column?: number } } };
            return emptyResult({
              message: e.message ?? 'SQL parse error',
              line: e.location?.start?.line,
              column: e.location?.start?.column,
            });
          }
          ```

       c. **Guard: single SELECT only**:
          `node-sql-parser` may return `Select | Select[]`. Normalize to a single AST. If result is an array with length > 1 → push `{ kind: 'non-select', reason: 'Multiple statements — only the first SELECT is imported' }` into `unsupportedConstructs` and take `ast[0]`. If `ast.type !== 'select'` → `unsupportedConstructs.push({ kind: 'non-select', reason: 'Only SELECT statements are supported' })` and bail early with an empty ParseResult + that construct.

       d. **extractColumns(ast)** — iterate `ast.columns`. For each entry:
          - If `col.expr.type === 'column_ref'` → normalize: strip wrapping `"..."`, uppercase, then `ALLOWED_COLUMNS.has(key)`. Match → push `{ key, alias: col.as ?? undefined, label: labelByKey.get(key) ?? key }` into `matchedColumns`. Unknown → `skippedColumns.push({ raw, reason: 'column not in schema' })`.
          - If `col.expr.type === 'aggr_func'` → `skippedColumns.push({ raw: col.expr.name ?? 'aggregate', reason: 'aggregate functions not supported — use raw columns' })` AND `unsupportedConstructs.push({ kind: 'aggregate', reason: 'GROUP BY / aggregate functions are not applied — imported as raw rows' })` (dedupe on kind).
          - Any other `expr.type` (case / function / expr_list) → `skippedColumns.push({ raw: JSON.stringify(col.expr).slice(0, 60), reason: \`expression type "${col.expr.type}" not supported\` })`.

       e. **extractFrom(ast)** — case-normalize `ast.from[0].table`; compare with `SOURCE_TABLE` from `@/lib/columns/config`. Mismatch → push `{ kind: 'table-mismatch', reason: \`Query references a different table: ${tableName} — columns are matched by name against app schema\` }` into `unsupportedConstructs`. No hard-fail. JOINs (`ast.from.length > 1` OR any `from` entry with a `join` field set) → `{ kind: 'join', reason: 'JOIN not supported — only the primary table is used' }`.

       f. **extractWhere(ast)** — call `flattenConjunction(ast.where)` helper that walks only AND nodes (Pitfall 5). For each leaf `binary_expr`:
          - Resolve LHS column key (same normalization as columns). If LHS is not a plain column_ref on an allow-listed column → `skippedFilters.push(...)`.
          - By `operator`:
            - `=` on text col → `{ op: 'eq' }`
            - `IN` (rhs is `expr_list`) → `{ op: 'in', value: values[] }`
            - `BETWEEN` → `{ op: 'between', value: { min, max } }`
            - `IS NULL` → `{ op: 'isNull', value: null }`  (v1: push into `skippedFilters` with reason `'IS NULL filters not yet supported'`; the TanStack filter schema has no null bucket).
            - Everything else (`LIKE`, `<`, `>`, `<>`, `NOT IN`, nested `binary_expr` with non-literal rhs) → `skippedFilters.push({ raw: printExpr(leaf), reason: 'operator not supported' })`.
          - Any top-level `OR` anywhere in the tree → `skippedFilters.push({ raw: printExpr(or_branch), reason: 'OR conditions not supported' })` (CONTEXT: "nested OR/AND beyond one level").

       g. **extractOrderBy(ast)** — for each `ast.orderby` entry: require `expr.type === 'column_ref'`, normalize key, allow-list check. Match → `matchedSort.push({ columnKey: key, desc: entry.type === 'DESC' })`. Unknown → `skippedSort.push({ raw, reason: 'column not in schema' })`.

       h. **Detect other unsupported constructs**:
          - CTE: presence of `ast.with` (non-null array) → `unsupportedConstructs.push({ kind: 'cte', reason: 'CTE (WITH clause) not supported — only the top-level SELECT is imported' })`.
          - Subquery in FROM: any `from[i].expr?.type === 'select'` → `{ kind: 'subquery', reason: 'Subquery in FROM not supported' }`.
          - GROUP BY: `ast.groupby && ast.groupby.length > 0` → `{ kind: 'groupby', reason: 'GROUP BY not supported — imported as raw rows' }`.
          - Window: any column with `expr.type === 'window_func'` → `{ kind: 'window', reason: 'Window functions not supported' }`.

       i. **Chart inference is lifted to Task 3** (`chart-inference.ts`) — `parseMetabaseSql` imports `inferChart(ast, matchedColumns, unsupportedConstructs)` and embeds the result under `inferredChart`.

       j. Export signature:
          ```typescript
          export function parseMetabaseSql(sql: string): ParseResult;
          ```
