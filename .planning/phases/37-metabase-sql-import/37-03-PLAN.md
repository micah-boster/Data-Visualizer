Wire the Import wizard's Apply button into the live app. Clicking Apply routes the parsed `ParseResult` + the original SQL string through a new `handleApplyImport` callback in `data-display.tsx` that:

1. Captures the current ViewSnapshot as the Undo target.

3. Calls `mapToSnapshot(result)` to get a `Partial<ViewSnapshot>`, merges with current defaults, stamps `sourceQuery: { sql, importedAt: Date.now() }`, and drives the table/chart via the existing `tableLoadViewRef` / `chartLoadRef` pipeline.

  <action>
    Implement the apply pipeline. This task is the business-logic heart of the phase тАФ budget accordingly.

    **Step A тАФ add imports at the top of `data-display.tsx`:**

    **Step B тАФ bind `handleApplyImport` immediately AFTER the existing `handleLoadView` callback (around line 409 тАФ right after its closing `[router, setActiveListId]` dependency array):**

    ```typescript
    // Phase 37 тАФ Apply path for Metabase SQL Import. Mirrors handleLoadView's
    // state-restore pipeline, but:
    //   1. Captures the pre-import ViewSnapshot for the Undo action BEFORE we
    //      mutate anything.
    //   2. Forces drill to root (CONTEXT lock + META-03) тАФ drill reset is
    //      unconditional, not snapshot-driven.
    //   3. Stamps sourceQuery: { sql, importedAt } on the working view so the
    //      audit-trail lands if the user later persists via Save View.
    //   4. Does NOT push into useSavedViews тАФ the imported view is an unsaved
    //      working view until the user uses the existing Save View flow.
    const handleApplyImport = useCallback(
      (result: ParseResult, sourceSql: string) => {
        // 1. Capture pre-import snapshot for Undo (same ref handleSaveView uses).
        const previousSnapshot = tableSnapshotRef.current?.();
        const previousChartsExpanded = chartsExpanded;
        const previousComparisonVisible = comparisonVisible;
        const previousDrill: DrillState = { ...drillState };

        // 2. Build a Partial<ViewSnapshot> from the parsed result, overlay on
        //    current-view defaults, stamp sourceQuery.
        const partial = mapToSnapshot(result);
        const base: ViewSnapshot = previousSnapshot ?? {
          sorting: [],
          columnVisibility: {},
          columnOrder: [],
          columnFilters: {},
          dimensionFilters: {},
          columnSizing: {},
        };

        const importedSnapshot: ViewSnapshot = {
          ...base,
          ...partial,
          // drill always cleared on import (CONTEXT lock).
          drill: undefined,
          // listId explicitly unset тАФ imports do NOT carry a partner list.
          listId: null,
          sourceQuery: { sql: sourceSql, importedAt: Date.now() },
        };

        // 3. Drill reset тАФ lifted verbatim from handleLoadView lines 370-383,
        //    without the snapshot.drill branch (we always clear).
        {
          const drillParams = new URLSearchParams(window.location.search);
          drillParams.delete('p');
          drillParams.delete('b');
          const drillQs = drillParams.toString();
          router.push(drillQs ? `?${drillQs}` : window.location.pathname, { scroll: false });
        }

        // 4. Chart state тАФ if the mapper produced a valid ChartDefinition AND
        //    it's a Phase-36 variant (line/scatter/bar), DataTable's chart
        //    refs will handle it once Phase 36 Wave 2+3 ship. Today, only
        //    collection-curve is wired via chartLoadRef. Imported charts are
        //    Phase-36 variants, so we set importedSnapshot.chartState and let
        //    it flow through the working view; the chart won't render until
        //    Phase 36 lands the dispatcher, at which point this just starts
        //    working. No code changes needed here.
        //
        //    Safety: if the mapper returns a collection-curve (it won't тАФ
        //    inferChart only emits line/scatter/bar), chartLoadRef still
        //    narrows correctly.
        if (importedSnapshot.chartState?.type === 'collection-curve' && chartLoadRef.current) {
          chartLoadRef.current(importedSnapshot.chartState);
        }

        // 5. Synthesize a working SavedView and drive through handleLoadView.
        //    id='working:import' is a sentinel тАФ useSavedViews never persists it
        //    because we don't call saveView. handleLoadView only READS from the
        //    SavedView shape.
        const workingView: SavedView = {
          id: 'working:import',
          name: 'Imported View (unsaved)',
          snapshot: importedSnapshot,
          createdAt: Date.now(),
        };

        // Apply dimension filters + table state via the existing ref.
        // (handleLoadView is already memoized; calling it is safe.)
        handleLoadView(workingView);

        // 6. Toast with summary + Undo action.
        const matchedCols = result.matchedColumns.length;
        const matchedFlts = result.matchedFilters.length;
        const skippedTotal =
          result.skippedColumns.length
          + result.skippedFilters.length
          + result.skippedSort.length
          + result.inferredChart.skipped.length
          + result.unsupportedConstructs.length;
        const description = `${matchedCols} column${matchedCols === 1 ? '' : 's'}, ${
          matchedFlts
        } filter${matchedFlts === 1 ? '' : 's'}${
          skippedTotal > 0 ? `, ${skippedTotal} skipped` : ''
        }`;

        toast('View imported', {
          description,
          action: {
            label: 'Undo',
            onClick: () => {
              // Restore via the same handleLoadView pipeline. Rebuild the
              // pre-import SavedView with the captured snapshot.
              if (!previousSnapshot) return;
              setChartsExpanded(previousChartsExpanded);
              setComparisonVisible(previousComparisonVisible);
              const restoreView: SavedView = {
                id: 'working:undo-import',
                name: 'Undo Import',
                snapshot: {
                  ...previousSnapshot,
                  chartsExpanded: previousChartsExpanded,
                  comparisonVisible: previousComparisonVisible,
                  drill: previousDrill.level === 'root'
                    ? undefined
                    : {
                        partner: previousDrill.partner ?? undefined,
                        batch: previousDrill.batch ?? undefined,
                      },
                },
                createdAt: Date.now(),
              };
              handleLoadView(restoreView);
            },
          },
          duration: 8000, // longer than save/delete (5s) тАФ import is higher-stakes
        });
      },
      [
        router,
        handleLoadView,
        chartsExpanded,
        comparisonVisible,
        drillState,
        setChartsExpanded,
        setComparisonVisible,
      ],
    );
    ```

    Pitfalls honoured:
    - **37-RESEARCH Pitfall (Apply pattern):** drill-reset recipe copied verbatim from lines 370-383. `router.push` with `{ scroll: false }` preserves scroll position and pushes a history entry (consistent with `handleLoadView`'s drill update pattern).
    - **Save is separate from Import (CONTEXT lock):** No `saveView(...)` call. The working view lives only in table/chart state + URL; persisting requires the user to use the existing Save View flow (the popover in the toolbar), at which point `sourceQuery` rides along automatically via `tableSnapshotRef.current()`.
    - **Legacy snapshot compat:** `sourceQuery` is optional; legacy views in localStorage parse cleanly (Plan 01 proved this).
    - **Fallback when `tableSnapshotRef.current` is null:** build `base` from an empty snapshot shape. This path triggers only if the table hasn't mounted yet тАФ extremely unlikely but non-crashing.
    - **Undo correctness:** the Undo onClick re-drives the same handleLoadView pipeline with the captured pre-import snapshot. Because `handleLoadView` re-writes URL params, the drill URL gets restored too.
