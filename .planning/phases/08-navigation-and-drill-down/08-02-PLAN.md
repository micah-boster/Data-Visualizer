Create four new files for the account data layer:

**src/lib/columns/account-config.ts** -- Column configuration for master_accounts.
- Follow the same `ColumnConfig` interface pattern from `src/lib/columns/config.ts`.
- Define `ACCOUNT_COLUMN_CONFIGS: ColumnConfig[]` with a curated ~18 columns:
  - Identity: PARTNER_NAME (text), BATCH (text), ACCOUNT_TYPE (text), STATUS (text)
  - Financial: TOTAL_BALANCE (currency), TOTAL_COLLECTED_ON_ACCOUNT (currency)
  - Payment: PAYMENT_PLAN_STATE (text), TOTAL_PAYMENTS (count or currency -- check schema)
  - Timing: ASSIGNMENT_DATE (date)
  - Demographics: US_STATE (text)
  - Additional useful columns (Claude's discretion): pick from the 78 available columns focusing on what a partnerships lead investigating a batch would care about. Include columns like balance bands, contact info flags, or resolution status if they exist. Target 15-20 columns total.
- Export `ACCOUNT_ALLOWED_COLUMNS: Set<string>` (derived from ACCOUNT_COLUMN_CONFIGS keys, same pattern as ALLOWED_COLUMNS in config.ts).
- Export `ACCOUNT_IDENTITY_COLUMNS: string[]` for columns that should appear in every view (PARTNER_NAME, BATCH, ACCOUNT_TYPE, STATUS).
- All columns defaultVisible: true (since this is already a curated set).

**src/lib/columns/account-definitions.ts** -- TanStack ColumnDef[] for accounts table.
- Follow the same pattern as `src/lib/columns/definitions.ts` which builds ColumnDef[] from COLUMN_CONFIGS.
- Import ACCOUNT_COLUMN_CONFIGS from account-config.ts.
- Reuse the same `getCellRenderer` and formatting utilities from `src/components/table/formatted-cell.tsx` and `src/lib/formatting/index.ts`.
- Export `accountColumnDefs: ColumnDef<Record<string, unknown>>[]`.
- Apply the same width, alignment, and formatting rules as the batch table columns.
- Pin PARTNER_NAME and BATCH to the left (same pattern as batch table).

**src/app/api/accounts/route.ts** -- New API route for master_accounts data.
- Export `dynamic = 'force-dynamic'`.
- GET handler accepts `partner` and `batch` query params (both required, return 400 if missing).
- Build SQL: `SELECT {columns} FROM master_accounts WHERE PARTNER_NAME = ? AND BATCH = ?` using `ACCOUNT_ALLOWED_COLUMNS` for the column list.
- CRITICAL: Use the `binds` parameter of `executeQuery` for the WHERE values. Never concatenate user input into SQL.
- Return the same `DataResponse` shape as the existing `/api/data` route for consistency.
- Wrap in try/catch with the same error handling pattern as `/api/data/route.ts` (log server-side, return safe error to client).

**src/hooks/use-account-data.ts** -- TanStack Query hook for account data.
- `useAccountData(partner: string | null, batch: string | null)`.
- Use `useQuery` with queryKey `['accounts', partner, batch]`.
- `enabled: !!partner && !!batch` (only fetch when both are set, which is batch drill level).
- Fetch from `/api/accounts?partner={encodeURIComponent(partner)}&batch={encodeURIComponent(batch)}`.
- Return `UseQueryResult<DataResponse>`.
