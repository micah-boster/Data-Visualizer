Create three new files:

**src/hooks/use-drill-down.ts** -- Drill-down state management hook.
- Export types: `DrillLevel = 'root' | 'partner' | 'batch'` and `DrillState { level, partner, batch }`.
- Read drill state from URL search params: `drillPartner` and `drillBatch` params. If both present -> level 'batch'. If only drillPartner -> level 'partner'. Otherwise -> 'root'.
- `drillToPartner(partnerName: string)`: Before navigating, capture all current non-drill search params (partner, type, batch filter params) and encode as base64 JSON string into a `drillFrom` param. Set `drillPartner` param. Use `router.push` (NOT router.replace) to create a browser history entry. This clears dimension filters per user decision.
- `drillToBatch(batchName: string)`: Add `drillBatch` to current params (preserving `drillPartner` and `drillFrom`). Use `router.push`.
- `navigateToLevel(level: DrillLevel)`: If 'root', decode `drillFrom` and restore those params (this restores pre-drill filters). If 'partner', remove `drillBatch` but keep `drillPartner` and `drillFrom`. Use `router.push` for all so back button works.
- CRITICAL: Use `router.push` for ALL drill navigation, never `router.replace`. This is the opposite of useFilterState which uses `router.replace`. The distinction is what makes browser back button work for drill-down.
- Use `useSearchParams`, `usePathname`, `useRouter` from `next/navigation`.
- Use `{ scroll: false }` option on all router.push calls to prevent scroll reset.

**src/components/navigation/breadcrumb-trail.tsx** -- Breadcrumb UI component.
- Props: `state: DrillState`, `rowCounts: { root?: number; partner?: number; batch?: number }`, `onNavigate: (level: DrillLevel) => void`.
- Always render root segment "All Batches" with count.
- If at partner level, add "Partner: {name}" segment.
- If at batch level, add both partner and batch segments.
- Active (current) segment: bold text, not clickable.
- Non-active segments: muted text, clickable button, hover:underline.
- Separator: ChevronRight icon from lucide-react (h-3.5 w-3.5).
- Row counts formatted with toLocaleString() and shown in parentheses.
- Wrap in `<nav aria-label="Drill-down breadcrumb">`.
- Position: between filter bar and table (handled in Task 2 wiring).

**src/components/navigation/drillable-cell.tsx** -- Clickable cell for partner name.
- Props: `value: string`, `onDrill: () => void`.
- Render as a `<button>` with text-primary, underline decoration, cursor-pointer, hover:text-primary/80.
- Click calls `onDrill()`. Only the partner name text is the click target (per user decision -- not entire row).
