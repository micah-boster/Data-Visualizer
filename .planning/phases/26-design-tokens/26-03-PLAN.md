- Surfaces: `bg-surface-raised` (raised), `bg-surface-base` (base page), `bg-surface-overlay` (dropdowns/popovers)

- Type: `text-title` (15px medium), `text-heading` (18px semibold)

  <action>
Read `src/components/layout/header.tsx` fully first. Map class replacements:

Outer shell (root element):
- `bg-background/80 backdrop-blur-sm` → `bg-surface-raised` (drop the blur for Phase 26; CONTEXT surface map classifies header as raised)
- `border-b border-border/50` → provisionally DROP and rely on `shadow-xs` for separation. If visual verification in Task 2 shows the header blends with the content area in either mode, re-add `border-b border-border` as a fallback.
- Add `shadow-xs` (subtlest multi-layer shadow; sells "this surface floats above base" without shouting)
- Hardcoded padding (e.g., `px-6 py-3` or similar) → `px-page-gutter py-stack` (semantic aliases — 16px horizontal, 12px vertical; matches CONTEXT semantic map). If these exact values feel wrong in context, use raw numeric tokens: `px-4 py-3` (`--spacing-4` = 16px, `--spacing-3` = 12px — same pixel values but via tokens).

Inner container / flex wrapper:
- Hardcoded `gap-*` values → `gap-stack` (12px) or `gap-inline` (8px) depending on content density. Pick whichever matches current visual.

Title text (page title or app name, whatever renders in header):
- Replace whatever size class exists (likely `text-lg font-semibold` or `text-xl`) with `text-title` (15px medium — per CONTEXT type ramp) OR `text-heading` (18px semibold) if the header title is the visual anchor of the screen. Read the existing size and pick the closest match.
- Drop standalone `font-semibold` / `font-medium` — the token's `--text-*--font-weight` handles it.

Any breadcrumb / sub-text:
- Use `text-label` (12px uppercase tracked) for eyebrow-style labels, or `text-caption` (12px) for subtle descriptive text, or `text-body` for default body.

Any buttons inside the header:
- Preserve their existing shadcn Button classes (shadcn Button uses ring-ring, bg-primary — all re-mapped via Plan 26-01). No changes needed beyond their existing className.
- The ThemeToggle button itself — do not modify; its styling comes from the shadcn Button primitive which now pulls from tokens automatically.

Focus ring:
- No focus-ring work in this plan. Existing `focus-visible:ring-*` classes on shadcn primitives continue to resolve via the Plan 26-01 re-map. Focus-ring is demoed and verified on `/tokens` (Plan 26-05) — keep scope tight here.

Dark mode:
- No explicit `dark:` variants needed. `bg-surface-raised` and `shadow-xs` both have `.dark` overrides in globals.css already. Remove any `dark:bg-*` utility on the header root (it will conflict with the token inversion where raised is LIGHTER than base in dark).

Backdrop blur:
- Plan: drop it entirely for Phase 26 (simpler, tokenized). If the user liked the blur, Phase 28 (Surfaces & Elevation) can reintroduce it as a `surface-raised--translucent` variant. Note this in the summary.

Do not change any JSX structure, event handlers, hook calls, or imports beyond class edits.
