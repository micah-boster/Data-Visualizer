Create `src/lib/formatting/` directory with four files:

**numbers.ts** -- Pure formatter functions using module-scoped `Intl.NumberFormat` instances (created once, reused):
- `formatCurrency(value: number): string` -- Uses `Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD', minimumFractionDigits: 2, maximumFractionDigits: 2 })`. Returns "$1,234.56", "-$1,234.56", "$0.00".
- `formatPercentage(value: number, decimals?: number): string` -- IMPORTANT: Data is 0-100 range (NOT 0-1). Do NOT use `Intl.NumberFormat` with `style: 'percent'` since it multiplies by 100. Use `value.toFixed(decimals ?? 1) + '%'`. For very small values: if `value > 0 && value < Math.pow(10, -(decimals ?? 1))` return `<${Math.pow(10, -(decimals ?? 1)).toFixed(decimals ?? 1)}%`. Handle negative mirror similarly.
- `formatCount(value: number): string` -- Uses `Intl.NumberFormat('en-US', { maximumFractionDigits: 0 })`. Returns "1,234,567".
- `formatNumber(value: number): string` -- Uses `Intl.NumberFormat('en-US', { maximumFractionDigits: 1 })`. Returns "1,234.5".
- `isNumericType(type: string): boolean` -- Returns true for 'currency', 'percentage', 'count', 'number'.
- `getFormatter(type: string): ((value: number) => string)` -- Returns the appropriate formatter for a column type. Falls back to `formatNumber`.

**dates.ts** -- Pure date formatter functions using module-scoped `Intl.DateTimeFormat` instances:
- `formatDate(value: string | Date): string` -- Uses `Intl.DateTimeFormat('en-US', { month: 'short', day: 'numeric', year: 'numeric' })`. Returns "Jan 15, 2024".
- `formatTimestamp(value: string | Date): string` -- Uses `Intl.DateTimeFormat('en-US', { month: 'short', day: 'numeric', year: 'numeric', hour: 'numeric', minute: '2-digit', timeZoneName: 'short' })`. For tooltip display on timestamp columns.

**thresholds.ts** -- Threshold configuration for conditional formatting:
- Export `ThresholdConfig` interface: `{ low?: { value: number; reason: string }; high?: { value: number; reason: string } }`.
- Export `COLUMN_THRESHOLDS: Record<string, ThresholdConfig>` keyed by column key. Include thresholds for: PENETRATION_RATE_POSSIBLE_AND_CONFIRMED (low: 5, high: 50), PENETRATION_RATE_CONFIRMED_ONLY (low: 5, high: 50), OUTBOUND_SMS_OPEN_RATE_FROM_DELIVERED (low: 10, high: 60), OUTBOUND_EMAIL_OPEN_RATE_FROM_DELIVERED (low: 5, high: 40), RAITO_FIRST_TIME_CONVERTED_ACCOUNTS (low: 3, high: 25), OUTBOUND_SMS_CLICK_RATE_FROM_OPENED (low: 2, high: 30), OUTBOUND_EMAIL_CLICK_RATE_FROM_OPENED (low: 1, high: 20), OUTBOUND_PHONE_VERIFY_RATE_FROM_ANSWERED (low: 10, high: 50), INBOUND_PHONE_VERIFY_RATE_FROM_ANSWERED (low: 10, high: 50). Each with a descriptive `reason` string like "Penetration rate below 5% threshold".
- Export `checkThreshold(value: number, config: ThresholdConfig): { isLow: boolean; isHigh: boolean; reason: string | null }`.
- Export `getThreshold(columnKey: string): ThresholdConfig | undefined` -- Looks up from COLUMN_THRESHOLDS.

Create `src/components/table/formatted-cell.tsx` -- A React component that wraps a formatted value with conditional styling.

Props interface:
```typescript
interface FormattedCellProps {
  value: unknown;            // Raw value from accessor
  formattedValue: string;    // Already formatted string from formatter
  type: string;              // Column type from config
  columnKey: string;         // Column key for threshold lookup
}
```

Component logic (in this priority order):
1. **Zero check:** If `typeof value === 'number' && value === 0`, render `<span className="text-[var(--cell-zero)]">{formattedValue}</span>`. This shows "$0.00" / "0.0%" / "0" in dimmed text.
2. **Negative check:** If `typeof value === 'number' && value < 0`, render `<span className="text-destructive">{formattedValue}</span>`. Red text using existing destructive color.
3. **Outlier check:** Call `getThreshold(columnKey)`. If a threshold exists, call `checkThreshold(value, threshold)`. If `isLow` or `isHigh`:
   - Wrap in `Tooltip` > `TooltipTrigger` > `TooltipContent` from `@/components/ui/tooltip`.
   - Apply background tint: `isLow` gets `bg-[var(--cell-tint-low)]`, `isHigh` gets `bg-[var(--cell-tint-high)]`.
   - Add `rounded px-1 -mx-1` for subtle padding around the tinted area.
   - TooltipContent shows the `reason` string from the threshold check.
   - IMPORTANT: Only render Tooltip for cells that actually have outlier status. Do NOT wrap non-outlier cells in Tooltip (performance per Research pitfall 5).
4. **Default:** Render `<>{formattedValue}</>` (no wrapper element needed).

Export `FormattedCell` as a named export.

Also export a helper function `getCellRenderer(type: string, columnKey: string, value: unknown): React.ReactNode` that:
1. Calls the appropriate formatter from `src/lib/formatting` based on `type` (using `getFormatter` for numeric types, `formatDate` for date type)
2. Returns `<FormattedCell value={value} formattedValue={formatted} type={type} columnKey={columnKey} />`
3. For `text` type, returns the raw value as-is (no FormattedCell wrapper)

The `getCellRenderer` function is what Plan 02 will use from column definitions.
